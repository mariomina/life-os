## Story

**As a** usuario con habilidades registradas y tiempo acumulado,
**I want** ver una vista de progreso por habilidad que muestre mi nivel actual, las horas invertidas y el avance hacia el siguiente nivel,
**so that** pueda entender de un vistazo qué tan cerca estoy de dominar cada competencia.

## Acceptance Criteria

### AC1 — Función pura `getSkillProgress`
**Given** una skill con `timeInvestedSeconds` y `level`,
**When** se llama a `getSkillProgress(skill)`,
**Then** retorna `{ currentLevel, nextLevel, progressPercent, hoursToNextLevel, totalHours }` donde:
- Umbrales: beginner (0-50h), intermediate (50-200h), advanced (200-500h), expert (500h+)
- `progressPercent`: % dentro del nivel actual (0-100)
- `hoursToNextLevel`: horas que faltan para el siguiente nivel (null si ya es expert)
- `totalHours`: horas totales invertidas

### AC2 — Skill level se actualiza automáticamente según timeInvestedSeconds
**Given** una skill donde `timeInvestedSeconds` supera el umbral del nivel actual,
**When** se llama a `computeSkillLevel(seconds)`,
**Then** retorna el nivel correcto según los umbrales (beginner/intermediate/advanced/expert).

### AC3 — Server Action `recalculateSkillLevel`
**Given** una skill con tiempo acumulado,
**When** se llama a `recalculateSkillLevel(skillId)`,
**Then** calcula el nivel correcto desde `timeInvestedSeconds` y actualiza `skills.level` si cambió,
Y retorna `{ success: true, levelChanged: boolean, newLevel? }`.

### AC4 — UI: `SkillProgressCard` en `/skills`
**Given** una skill con tiempo invertido,
**When** se renderiza la lista de /skills,
**Then** cada skill muestra una tarjeta de progreso con:
- Nombre de la skill + badge de nivel actual
- Barra de progreso visual (0-100% hacia el siguiente nivel)
- Texto "X horas · faltan Y horas para [Next Level]" (o "Nivel máximo" si expert)
- Área vinculada (si existe)

### AC5 — UI: filtros y ordenamiento en Skills list
**Given** el usuario está en /skills,
**When** usa los controles de filtro,
**Then** puede filtrar por nivel (All / Beginner / Intermediate / Advanced / Expert)
Y ordenar por: Nombre (A-Z), Nivel (asc/desc), Tiempo invertido (desc).

### AC6 — Función pura `computeSkillLevel`
**Given** un número de segundos,
**When** se llama a `computeSkillLevel(seconds)`,
**Then** retorna el nivel correcto:
- < 180000s (50h) → 'beginner'
- 180000-719999s (50-199.9h) → 'intermediate'
- 720000-1799999s (200-499.9h) → 'advanced'
- ≥ 1800000s (500h+) → 'expert'

### AC7 — `recalculateSkillLevel` llamado tras cada `stopTimer`
**Given** el timer para y hay skills etiquetadas en la activity,
**When** `stopTimer` ya actualizó `timeInvestedSeconds` (Story 7.2),
**Then** también llama a `recalculateSkillLevel(skillId)` para cada skill actualizada,
Asegurando que el nivel en DB siempre es coherente con el tiempo.

### AC8 — Tests ≥ 556 + nuevos de Story 7.4
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan (≥ 556 anteriores + nuevos de Story 7.4).

## Risks

- **Nivel en DB vs nivel calculado:** Si `timeInvestedSeconds` aumentó pero `level` en DB no fue actualizado (ej: bug en Story 7.2), `recalculateSkillLevel` actúa como safety net. Llamarlo también cuando se carga /skills como check opcional.
- **Expert con progress 100%:** Si la skill es `expert`, la barra muestra 100% y el texto "Nivel máximo" — sin "faltan X horas".
- **0h tiempo:**  Si `timeInvestedSeconds = 0`, la barra muestra 0% y la skill está en beginner.

## Dev Notes

### Umbrales de nivel

```typescript
// features/skills/level.ts

export const SKILL_THRESHOLDS_SECONDS = {
  beginner:     0,
  intermediate: 50  * 3600,   // 180 000s
  advanced:     200 * 3600,   // 720 000s
  expert:       500 * 3600,   // 1 800 000s
} as const

export type SkillLevel = 'beginner' | 'intermediate' | 'advanced' | 'expert'
export const LEVEL_ORDER: SkillLevel[] = ['beginner', 'intermediate', 'advanced', 'expert']

export function computeSkillLevel(seconds: number): SkillLevel {
  if (seconds >= SKILL_THRESHOLDS_SECONDS.expert) return 'expert'
  if (seconds >= SKILL_THRESHOLDS_SECONDS.advanced) return 'advanced'
  if (seconds >= SKILL_THRESHOLDS_SECONDS.intermediate) return 'intermediate'
  return 'beginner'
}
```

### `getSkillProgress`

```typescript
export interface SkillProgress {
  currentLevel: SkillLevel
  nextLevel: SkillLevel | null
  progressPercent: number     // 0-100 within current level
  hoursToNextLevel: number | null
  totalHours: number
}

export function getSkillProgress(timeInvestedSeconds: number): SkillProgress {
  const totalHours = timeInvestedSeconds / 3600
  const currentLevel = computeSkillLevel(timeInvestedSeconds)
  const currentIdx = LEVEL_ORDER.indexOf(currentLevel)
  const nextLevel = currentIdx < LEVEL_ORDER.length - 1 ? LEVEL_ORDER[currentIdx + 1] : null

  if (!nextLevel) {
    return { currentLevel, nextLevel: null, progressPercent: 100, hoursToNextLevel: null, totalHours }
  }

  const levelStart = SKILL_THRESHOLDS_SECONDS[currentLevel]
  const levelEnd = SKILL_THRESHOLDS_SECONDS[nextLevel]
  const progressInLevel = timeInvestedSeconds - levelStart
  const levelRange = levelEnd - levelStart
  const progressPercent = Math.min(100, Math.floor((progressInLevel / levelRange) * 100))
  const secondsToNext = levelEnd - timeInvestedSeconds
  const hoursToNextLevel = Math.ceil(secondsToNext / 3600)

  return { currentLevel, nextLevel, progressPercent, hoursToNextLevel, totalHours }
}
```
