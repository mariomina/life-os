## Story

**As a** usuario que gestiona su tiempo diario,
**I want** ver en la Vista Día un timeline real de mis actividades programadas con su Time Budget y botones de acción rápida,
**so that** pueda ejecutar mi día desde el calendario sin tener que navegar a otras pantallas.

## Acceptance Criteria

### AC1 — Query `getActivitiesForDay`

**Given** un usuario autenticado y una fecha,
**When** se llama `getActivitiesForDay(userId, date)`,
**Then:**
- Devuelve todas las `steps_activities` del usuario donde `scheduledAt` cae dentro del día UTC especificado (00:00:00Z – 23:59:59.999Z)
- Incluye `areaName` (LEFT JOIN `areas`) y `habitTitle` (LEFT JOIN `habits`) — `null` si no aplica
- Incluye el campo `color` derivado de `areas.color` (o `'blue'` por defecto si es null)
- Ordena por `scheduledAt ASC`
- Está ubicada en `lib/db/queries/calendar.ts`
- Tiene tests unitarios con mocks de Drizzle

### AC2 — `page.tsx` consume datos reales

**Given** el calendario carga la Vista Día,
**When** el Server Component renderiza,
**Then:**
- `app/(app)/calendar/page.tsx` llama `getActivitiesForDay(user.id, new Date())` para el día actual
- Convierte el resultado con `toCalendarEvents()` de `calendar-utils.ts`
- Pasa `events` reales al `CalendarClient` (elimina los `STUB_EVENTS`)
- Si la query falla, renderiza el calendario con `events=[]` sin romper la página (error silencioso logueado)

### AC3 — DayView con timezone local correcto

**Given** la Vista Día muestra slots de horas,
**When** un evento está programado a las 09:00 UTC y el usuario está en UTC-6,
**Then:**
- El evento aparece en el slot de las **03:00** local (la hora correcta para ese timezone)
- La comparación de slot usa `e.start.getHours()` (hora local) en lugar de `e.start.getUTCHours()` — corrige el bug reportado en Story 5.1 QA (observación #1)
- La misma corrección aplica en `WeekView`

### AC4 — Time Budget panel en Vista Día

**Given** el usuario está en la Vista Día,
**When** hay actividades programadas para ese día,
**Then:**
- Bajo el toolbar del calendario aparece un panel "Time Budget" con:
  - **Comprometido:** suma de `scheduledDurationMinutes` de las actividades del día, en formato `Xh Ym`
  - **Disponible:** `16h 0m` (jornada activa estándar: 6:00–22:00 = 960 min)
  - **Libre:** diferencia entre disponible y comprometido (puede ser negativa → mostrar en rojo)
- El panel solo se muestra en Vista Día (`view === 'day'`)
- Si no hay actividades, muestra `0h 0m` comprometido y `16h 0m` libre

### AC5 — Botones de acción directa en eventos

**Given** un evento está visible en el timeline de la Vista Día,
**When** el usuario lo ve,
**Then:**
- Cada tarjeta de evento muestra (al hacer hover en desktop / siempre en móvil):
  - **"Iniciar"** — botón primario (solo si el evento no tiene `status: 'done'` y no hay un `timeEntry.isActive` asociado al evento)
  - **"Check-in"** — botón secundario (navega a `/checkin` con `activityId` como query param)
  - **"Omitir"** — botón terciario ghost (solo visible en hover)
- Los tres botones son elementos `<button>` accesibles con `aria-label`
- En esta Story, "Iniciar" no inicia el timer real (eso es Story 5.8) — en cambio, muestra un `toast` de "Timer — próximamente en Story 5.8"
- "Omitir" no implementa la lógica de skip (Story 5.7) — muestra toast similar
- "Check-in" sí navega correctamente

### AC6 — Vista Día muestra hora en timezone local

**Given** el usuario visualiza el timeline,
**When** los slots de horas se renderizan,
**Then:**
- Los labels de hora (e.g., "09:00") muestran la hora en **timezone local** del navegador
- Se usa `format(hour, 'HH:mm')` de `date-fns` (ya implementado en Story 5.1 — confirmar que es correcto)
- El rango de slots visible en Vista Día es 07:00–22:00 (15 slots)

### AC7 — Estado de actividad refleja colores de área

**Given** una actividad tiene un área asociada con color,
**When** se muestra en el timeline,
**Then:**
- El color de la tarjeta del evento refleja el color del área (mapeado a `TEventColor`)
- Si el área no tiene color reconocido, se usa `'blue'` como fallback
- El mapeo area.color → TEventColor se hace en `getActivitiesForDay` o en el Server Component (no en el Client)

### Time Budget cálculo

```typescript
const AVAILABLE_MINUTES = 16 * 60 // 6:00–22:00

function calcTimeBudget(events: ICalendarEvent[]) {
  const committed = events.reduce((acc, e) => {
    return acc + (e.end.getTime() - e.start.getTime()) / 60000
  }, 0)
  return {
    committed,
    available: AVAILABLE_MINUTES,
    free: AVAILABLE_MINUTES - committed,
  }
}
```
