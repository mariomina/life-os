## Story

**As a** usuario con métricas y correlaciones calculadas,
**I want** recibir insights en lenguaje natural generados por IA sobre mis patrones,
**so that** pueda entender el significado de los datos sin interpretar coeficientes estadísticos manualmente.

## Acceptance Criteria

### AC1 — Interface `ILLMProvider` (crear si no existe)
**Given** la necesidad de IA agnóstica de proveedor (FR15),
**When** se define la interfaz,
**Then** `ILLMProvider` tiene al menos:
```typescript
interface ILLMProvider {
  generateInsight(prompt: string): Promise<string>
}
```
Y se implementa `ClaudeProvider` (usando `@anthropic-ai/sdk`) como proveedor por defecto,
Con fallback graceful: si la API falla, retornar mensaje estático descriptivo (no error).

### AC2 — Función pura `buildInsightPrompt`
**Given** un array de correlaciones y métricas de resumen,
**When** se llama a `buildInsightPrompt(correlations, summary)`,
**Then** retorna un prompt estructurado en español para el LLM que incluye:
- Resumen de métricas (CCR, habit consistency, OKR progress)
- Correlaciones significativas (type, entities, coefficient)
- Instrucción: "Genera 3 insights concretos en 2-3 frases cada uno. Sé específico con los datos."

### AC3 — Server Action `generateReportInsights()`
**Given** el usuario autenticado con datos de 8.2 + correlaciones de 8.3,
**When** se llama a `generateReportInsights()`,
**Then**:
1. Obtiene métricas (CCR, habit consistency avg, OKR progress avg)
2. Obtiene correlaciones activas con tier='full'
3. Construye prompt con `buildInsightPrompt`
4. Llama al ILLMProvider
5. Retorna `{ insights: string, generatedAt: Date, provider: string }`

### AC4 — Fallback graceful (FR22)
**Given** el ILLMProvider no disponible (API error, timeout, key no configurada),
**When** se llama a `generateReportInsights()`,
**Then** retorna un mensaje estático descriptivo basado solo en los datos numéricos,
SIN lanzar error al usuario.
Ejemplo: "Tu CCR esta semana fue 73%. Mantuviste 4 de 5 hábitos. No hay correlaciones con datos suficientes aún."

### AC5 — UI: sección "Insights IA" en `/reports`
**Given** la página `/reports`,
**When** se renderiza la sección Insights,
**Then** aparece sección con:
- Texto de insights generado por IA (en markdown renderizado)
- Indicador de cuándo se generó ("Generado hace X minutos")
- Botón "Regenerar" (llama `generateReportInsights` de nuevo)
- Si el proveedor no está configurado: banner informativo con instrucciones para configurar API key

### AC6 — Configuración via env var
**Given** la arquitectura agnóstica (FR15),
**When** se lee la configuración,
**Then** el proveedor se selecciona por `NEXT_PUBLIC_LLM_PROVIDER` env var (`claude | openai | mock`).
`mock` retorna texto estático predefinido — útil para desarrollo y tests.

### AC7 — Tests ≥ anteriores + nuevos de Story 8.6
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan (funciones puras testeadas, LLM mockeado en tests).

## Business Value

Transforma datos crudos en lenguaje accionable. Sin insights NL, el usuario necesita interpretar coeficientes — con insights NL, entiende directamente "haces más ejercicio los lunes cuando tienes menos reuniones". Esta es la pieza que hace el sistema verdaderamente inteligente para el usuario no técnico.

## Risks

- **Costo de API:** Cada `generateReportInsights` hace 1 llamada Claude. Para MVP single-user es negligible.
- **Latencia:** La llamada LLM puede tardar 2-5 segundos. Mostrar loading state.
- **API key no configurada:** El fallback estático debe ser informativo, no un error de stack.
- **Prompt injection:** El contenido de correlaciones viene de la DB del usuario (datos seguros). Aun así, sanitizar nombres de entidades antes de incluirlos en el prompt.

## Dev Notes

### `lib/llm/interface.ts`

```typescript
export interface ILLMProvider {
  generateInsight(prompt: string): Promise<string>
  readonly providerName: string
}
```

### `lib/llm/claude.ts`

```typescript
import Anthropic from '@anthropic-ai/sdk'

export class ClaudeProvider implements ILLMProvider {
  readonly providerName = 'claude'
  private client: Anthropic

  constructor() {
    this.client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })
  }

  async generateInsight(prompt: string): Promise<string> {
    const msg = await this.client.messages.create({
      model: 'claude-haiku-4-5-20251001',
      max_tokens: 500,
      messages: [{ role: 'user', content: prompt }],
    })
    return msg.content[0].type === 'text' ? msg.content[0].text : ''
  }
}
```

### `lib/llm/mock.ts`

```typescript
export class MockProvider implements ILLMProvider {
  readonly providerName = 'mock'
  async generateInsight(_prompt: string): Promise<string> {
    return 'Análisis disponible. Mantuviste una consistencia sólida esta semana. Continúa con tus hábitos actuales.'
  }
}
```

### Factory

```typescript
// lib/llm/factory.ts
export function getLLMProvider(): ILLMProvider {
  const provider = process.env.NEXT_PUBLIC_LLM_PROVIDER ?? 'mock'
  if (provider === 'claude') return new ClaudeProvider()
  return new MockProvider()
}
```

### `buildInsightPrompt` sketch

```typescript
// features/reports/insights.ts
export function buildInsightPrompt(summary: MetricsSummary, correlations: Correlation[]): string {
  const significantCorrelations = correlations
    .filter(c => c.tier === 'full' && c.type !== 'neutral')
    .slice(0, 5)
  return `Analiza los siguientes datos del usuario y genera 3 insights concretos en español...`
}
```
