### Patrones arquitecturales críticos

[Source: docs/architecture/coding-standards.md#12.3]

- **`'use client'` mínimo:** Solo en componentes con interactividad. Server Components por defecto.
- **Path alias:** `@/` mapea a la raíz del proyecto (ya configurado en tsconfig.json)
- **Imports absolutos:** Usar `@/components/...`, `@/lib/...`, `@/features/...` — nunca rutas relativas multi-nivel
- **Naming:** Componentes React = PascalCase; Server Actions = camelCase verbo-primero; DB functions = `get/create/update/delete` prefix

### Notas sobre dark mode con Tailwind v4

[Source: docs/architecture/tech-stack.md + docs/prd/technical-assumptions.md]

Tailwind CSS 4 cambia el sistema de dark mode. Usar estrategia `class` en `tailwind.config.ts`:

```typescript
// tailwind.config.ts (si se requiere config explícita en v4)
// En Tailwind v4 el dark mode es via @media o class — TailAdmin 2.2.2 usa class
```

Verificar que TailAdmin 2.2.2 ya viene preconfigurado para Tailwind v4 — no asumir que la configuración de Tailwind v3 funciona directamente.
