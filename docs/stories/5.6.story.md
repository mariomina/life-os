## Story

**As a** usuario que quiere ver sus compromisos futuros,
**I want** ver en la Vista Agenda una lista cronológica de las actividades de los próximos 30 días con su hora, duración y área,
**so that** pueda tener visibilidad rápida de mis compromisos sin navegar vista a vista y planificar mejor mis próximos días.

## Acceptance Criteria

### AC1 — AgendaView muestra datos reales del año

**Given** el usuario está en la Vista Agenda y hay actividades en `steps_activities`,
**When** el componente renderiza,
**Then:**
- La Vista Agenda ya recibe los eventos del año completo desde `page.tsx` (via `getActivitiesForYear`) — sin query adicional
- Muestra las actividades de los próximos 30 días desde la fecha actual (`currentDate`) agrupadas por día
- Cada entrada muestra: hora de inicio (HH:mm), título, y área (`description` del evento = `areaName`) si está disponible
- Cada entrada muestra la duración en minutos (calculada como `end - start` en minutos, formato `Xm` o `Xh Ym`)
- El color de área aparece como borde izquierdo coloreado (ya implementado con `EVENT_COLOR_CLASSES`)
- Si no hay eventos en los próximos 30 días, muestra "No hay eventos en los próximos 30 días"

### AC2 — AgendaView muestra duración de cada evento

**Given** el usuario está en la Vista Agenda,
**When** un evento tiene `start` y `end` definidos,
**Then:**
- La duración se muestra junto a la hora: `HH:mm · Xm` o `HH:mm · Xh Ym`
- Duración < 60 min → `Xm` (ej: `09:00 · 30m`)
- Duración >= 60 min → `Xh Ym` (ej: `09:00 · 1h 30m`)
- Reutiliza `formatMinutes()` exportada de `CalendarClient.tsx` (sin duplicar lógica)

### AC3 — AgendaView muestra el área del evento

**Given** el usuario está en la Vista Agenda y un evento tiene `description` (= `areaName`),
**When** el componente renderiza esa fila,
**Then:**
- El nombre del área aparece como badge/tag pequeño junto al título
- Si `description` es `undefined` (no aplica), no muestra nada adicional (sin "undefined" visible)
- El tag usa el mismo color de borde que la fila del evento

### AC4 — Navegación en Vista Agenda avanza/retrocede por meses

**Given** el usuario está en la Vista Agenda y usa los botones de navegación,
**When** presiona "siguiente" o "anterior",
**Then:**
- La vista avanza/retrocede 1 mes (ya implementado: `addMonths`/`subMonths` en `navigate`)
- Muestra los próximos 30 días desde la nueva `currentDate` — sin nueva query (filtra del array del año)
- Al presionar "Hoy" regresa al día actual

### AC5 — Tests: comportamiento de AgendaView

**Given** la implementación completa,
**When** se ejecutan los tests,
**Then:**
- Se agrega `tests/unit/calendar/agenda-view.test.ts` con al menos 5 casos usando las funciones puras de calendar-utils:
  1. Días con eventos aparecen en la agenda (usando `getEventsForDay`)
  2. Días sin eventos no aparecen en la lista
  3. Próximos 30 días — día 31 excluido
  4. Estado vacío: `daysWithEvents.length === 0`
  5. Duración formateada correctamente: `formatMinutes` — `30m`, `60m → 1h 0m`, `90m → 1h 30m`
- Todos los tests anteriores pasan sin regresión (343+ → 348+ tests)
- CI: lint 0 errores, typecheck 0 errores, build ✅

### Exportar formatMinutes

```typescript
// ANTES: function formatMinutes(minutes: number): string {
// DESPUÉS:
export function formatMinutes(minutes: number): string {
  // misma implementación — solo agregar export
}
```

**Nota:** El comportamiento de `formatMinutes` con valores < 60 retorna `Xh 0m` — para Agenda, filtrar casos < 60 y mostrar solo `Xm` sin la parte de horas. Verificar si se prefiere:
- **Opción A:** `30m` para < 60min, `1h 30m` para >= 60min (más legible en Agenda)
- **Opción B:** Siempre `Xh Ym` (consistente con TimeBudgetPanel)

**Recomendación:** Opción A para Agenda — más natural en una lista de compromisos.

```typescript
export function formatMinutes(minutes: number): string {
  const abs = Math.abs(minutes)
  const h = Math.floor(abs / 60)
  const m = Math.round(abs % 60)
  if (h === 0) return `${m}m`
  return `${h}h ${m}m`
}
```

⚠️ **Este cambio afecta `TimeBudgetPanel` y `WeeklyTimeBudgetPanel`** — verificar que siguen funcionando correctamente. Si `TimeBudgetPanel` espera siempre `Xh Ym`, mantener la función original y crear una versión específica para Agenda.

**Decisión del @dev:** Mantener `formatMinutes` original (`Xh Ym` siempre) para los Time Budget panels y crear función local `formatDuration` en `AgendaView` para mostrar `Xm`/`Xh Ym`.

### Tests — patrón

```typescript
// tests/unit/calendar/agenda-view.test.ts
import { describe, it, expect } from 'vitest'
import { getEventsForDay } from '@/lib/calendar/calendar-utils'
import type { ICalendarEvent } from '@/lib/calendar/calendar-utils'
import { addDays, eachDayOfInterval } from 'date-fns'

// Helper: simula la lógica de AgendaView
function getAgendaDays(currentDate: Date, events: ICalendarEvent[]) {
  const next30Days = eachDayOfInterval({
    start: currentDate,
    end: addDays(currentDate, 29),
  })
  return next30Days
    .map((day) => ({ day, events: getEventsForDay(events, day) }))
    .filter(({ events }) => events.length > 0)
}
```
