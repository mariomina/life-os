## Story

**As a** usuario nuevo que eligió "Cuestionario guiado" en el onboarding,
**I want** responder 5 preguntas por cada una de las 8 áreas Maslow con escala Likert 1-5,
**so that** el sistema calcule mi score inicial por área (0-100%) y mi Life System Health Score global, guarde esos datos en DB, y me redirija al Home con mi baseline establecido.

## Scope

**IN scope:**
- Reemplazar el placeholder de `/onboarding/questionnaire/page.tsx` con el cuestionario real
- Cuestionario multi-step: una área por pantalla, 5 preguntas Likert 1-5 por área (total 8 pantallas de preguntas)
- Preguntas basadas en las dimensiones de cada área Maslow (hardcoded en código — no en DB para MVP)
- Escala Likert 1-5 con etiquetas: "Nunca" / "Raramente" / "A veces" / "Frecuentemente" / "Siempre"
- Cálculo de score por área: `(suma_respuestas - 5) / 20 × 100` → escala 0-100 (5 preguntas × escala 1-5)
- Al completar las 8 áreas:
  1. Crear las 8 filas en tabla `areas` (seed de las 8 áreas Maslow para el usuario)
  2. Insertar `area_scores` inicial por cada área con el score calculado
  3. Actualizar `areas.current_score` con el score calculado
  4. Marcar `user_metadata.onboarding_completed = true` en Supabase Auth
  5. Redirect a `/` (Home)
- Progress bar mostrando área actual (1 de 8, 2 de 8, etc.)
- Botones Anterior/Siguiente entre áreas — respuestas se conservan al volver
- Pantalla de resumen final antes de guardar: muestra score % por área y Life System Health Score global
- Server Actions: `seedUserAreas(userId)`, `saveAreaScores(userId, scores[])`, `completeOnboarding(userId)`
- Drizzle ORM para writes — usando `lib/db/client.ts` existente
- TypeScript strict, sin `any`
- Tests unitarios para: función de cálculo de score por área, función de cálculo del score global desde respuestas

**OUT of scope:**
- Preguntas en DB (hardcoded para MVP — Story futura si se necesita admin de preguntas)
- Cuestionario adaptativo (mismas 5 preguntas por área para todos)
- Validación psicométrica formal de las preguntas (basadas en dimensiones del PRD)
- Editar respuestas después de completar (Story 2.5 o futura)
- Multi-idioma
- Animaciones de transición entre áreas

### Fórmula de score por área desde Likert

```typescript
// 5 preguntas × escala 1-5
// Mínimo posible: 5 (todas en 1) → 0%
// Máximo posible: 25 (todas en 5) → 100%
function calculateAreaScoreFromResponses(responses: number[]): number {
  const sum = responses.reduce((a, b) => a + b, 0)
  return ((sum - 5) / 20) * 100
}
// Ejemplos:
// [1,1,1,1,1] → (5-5)/20 × 100 = 0%
// [5,5,5,5,5] → (25-5)/20 × 100 = 100%
// [3,3,3,3,3] → (15-5)/20 × 100 = 50%
```

### localStorage para estado del wizard

```typescript
// En el componente del cuestionario (Client Component)
const STORAGE_KEY = 'life-os-questionnaire-state'

// Guardar al cambiar de área
useEffect(() => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify({ currentArea, responses }))
}, [currentArea, responses])

// Restaurar al montar
useEffect(() => {
  const saved = localStorage.getItem(STORAGE_KEY)
  if (saved) {
    const { currentArea: savedArea, responses: savedResponses } = JSON.parse(saved)
    setCurrentArea(savedArea)
    setResponses(savedResponses)
  }
}, [])
```
