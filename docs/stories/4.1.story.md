## Story

**As a** usuario con áreas y OKRs definidos,
**I want** poder crear, ver, editar y cancelar proyectos vinculados a un área y opcionalmente a un KR,
**so that** pueda organizar mi ejecución en vehículos concretos con nombre, descripción, estado y contexto Maslow visible.

## Dependencies

- **Requires:** Story 1.4 — Done ✅ (tabla `projects` con `area_id`, `okr_id`, `status`, `template_id` en schema Drizzle)
- **Requires:** Story 3.5 — Done ✅ (OKRs anuales disponibles para vincular)
- **Requires:** Story 3.1 — Done ✅ (áreas disponibles con nombre y nivel Maslow)
- **Blocks:** Story 4.2 (Visual Workflow Builder necesita proyectos como entidad padre)
- **Blocks:** Story 4.3 (Templates requieren proyecto donde instanciarse)

## Scope

### IN

- **Página `/projects`:** lista de proyectos activos del usuario (Server Component)
- **Crear proyecto:** formulario inline con título (required), descripción (opcional), área (required — selector con nombre + nivel Maslow), KR (opcional — selector filtrado por área seleccionada), estado inicial = `active`
- **Ver detalle:** `/projects/[id]` — muestra título, descripción, área, KR vinculado, estado, fecha creación
- **Editar proyecto:** actualizar título, descripción, área, KR vinculado desde página de detalle
- **Cancelar proyecto:** delete lógico → `status = 'archived'` (no borrado físico)
- **Filtro por estado:** tabs `Activos | Pausados | Completados | Archivados` en la lista
- **Server Actions:** `createProject`, `updateProject`, `archiveProject` en `actions/projects.ts`
- **Query layer:** `getProjects`, `getProjectById` en `lib/db/queries/projects.ts`
- **Tests unitarios:** lógica de validación de creación (título requerido, área requerida)

### OUT

- Visual Workflow Builder dentro del proyecto (Story 4.2)
- Templates de workflow (Story 4.3)
- Squads (Story 4.4)
- Completar proyecto (status `completed`) — lo maneja Story 4.5+
- Métricas de tiempo por proyecto (Story 8.1)

## Acceptance Criteria

**AC1 — Lista de proyectos:**
Given el usuario está en `/projects`,
When la página carga,
Then ve una lista de todos sus proyectos con: título, área vinculada (nombre + badge nivel Maslow), KR vinculado (título, si existe), estado (badge coloreado), y fecha de creación.

**AC2 — Crear proyecto:**
Given el usuario hace click en "Nuevo Proyecto",
When completa título + selecciona área y hace submit,
Then el proyecto se crea con `status = 'active'`, aparece en la lista sin recargar la página, y el formulario se cierra.

**AC3 — Título requerido:**
Given el usuario intenta crear un proyecto sin título,
Then el formulario muestra error "El título es requerido" y no hace submit.

**AC4 — Área requerida:**
Given el usuario intenta crear un proyecto sin seleccionar área,
Then el formulario muestra error "El área es requerida" y no hace submit.

**AC5 — KR filtrado por área:**
Given el usuario selecciona un área en el formulario,
When el selector de KR se activa,
Then solo muestra OKRs anuales y KRs activos vinculados a esa área.

**AC6 — Editar proyecto:**
Given el usuario está en `/projects/[id]`,
When actualiza título o descripción o área o KR y guarda,
Then los cambios persisten y se muestran actualizados.

**AC7 — Archivar proyecto:**
Given el usuario hace click en "Archivar" con confirmación,
Then el proyecto cambia a `status = 'archived'`, desaparece de la tab "Activos", y aparece en "Archivados".

**AC8 — Filtro por estado:**
Given el usuario está en `/projects`,
When selecciona una tab (Activos / Pausados / Completados / Archivados),
Then la lista muestra solo proyectos con ese estado.

**AC9 — Proyecto sin KR:**
Given el usuario crea un proyecto sin KR vinculado,
Then el proyecto se crea correctamente y muestra "Sin KR vinculado" en el detalle.

**AC10 — Datos del OKR en lista:**
Given un proyecto está vinculado a un KR,
When aparece en la lista,
Then muestra el título del KR (no el ID) en formato readable.

### Selector de KR filtrado por área
Al seleccionar área en `ProjectForm`, hacer fetch client-side de KRs activos para esa área. Usar la query existente de OKRs pero filtrando por `areaId` y `type = 'key_result'` y `status = 'active'`.
