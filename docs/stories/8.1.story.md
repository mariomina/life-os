## Story

**As a** usuario con activities y time entries registrados,
**I want** ver cuánto tiempo he invertido por área y por proyecto en distintos períodos,
**so that** pueda entender cómo distribuyo mi tiempo y tomar decisiones informadas.

## Acceptance Criteria

### AC1 — Server Action `getTimeByArea`
**Given** el usuario autenticado y un período (`week | month | quarter`),
**When** se llama a `getTimeByArea(period)`,
**Then** retorna `{ areaId: string, areaName: string, totalSeconds: number }[]` ordenado por `totalSeconds` desc,
Filtrando `time_entries` cuyo `created_at` cae dentro del período seleccionado.

### AC2 — Server Action `getTimeByProject`
**Given** el usuario autenticado y un período,
**When** se llama a `getTimeByProject(period)`,
**Then** retorna `{ projectId: string, projectName: string, totalSeconds: number }[]` ordenado por `totalSeconds` desc,
Usando JOIN: `time_entries → steps_activities → tasks → workflows → projects`.
Activities sin proyecto retornan bajo `projectId: 'none', projectName: 'Sin proyecto'`.

### AC3 — Lógica de períodos
**Given** un período `week | month | quarter`,
**When** se calcula el rango de fechas,
**Then**:
- `week`: últimos 7 días (now - 7d)
- `month`: últimos 30 días (now - 30d)
- `quarter`: últimos 90 días (now - 90d)

### AC4 — Página `/reports` con sección Time by Area
**Given** el usuario navega a `/reports`,
**When** se renderiza la página,
**Then** aparece sección "Tiempo por Área" con tabla: nombre de área + tiempo formateado (`Xh Ymin`) + barra visual relativa (% del total).

### AC5 — Sección "Tiempo por Proyecto" en `/reports`
**Given** la página `/reports`,
**When** se renderiza,
**Then** aparece sección "Tiempo por Proyecto" con tabla: nombre de proyecto + tiempo + barra visual.

### AC6 — Selector de período

**When** el usuario cambia el período (Semana/Mes/Trimestre),

### AC7 — Estado vacío

## Business Value

## Risks

- **JOIN chain profundo (time→activity→task→workflow→project):** Puede ser nulo en cualquier nivel. Usar LEFT JOIN y agrupar actividades sin proyecto bajo "Sin proyecto".
- **Período vacío:** Si no hay time entries en el período, las queries retornan arrays vacíos — manejar estado vacío en UI.
- **`durationSeconds` null:** Solo sumar rows con `durationSeconds IS NOT NULL` (timers detenidos).

## Dev Notes

### `getPeriodRange(period)`

```typescript
// features/reports/periods.ts
export type ReportPeriod = 'week' | 'month' | 'quarter'

export function getPeriodRange(period: ReportPeriod, now = new Date()): { from: Date; to: Date } {
  const to = now
  const from = new Date(now)
  if (period === 'week') from.setDate(from.getDate() - 7)
  else if (period === 'month') from.setDate(from.getDate() - 30)
  else from.setDate(from.getDate() - 90)
  return { from, to }
}
```

### `getTimeByArea` (sketch)

```typescript
// actions/reports.ts
import { getPeriodRange } from '@/features/reports/periods'
import { timeEntries } from '@/lib/db/schema/time-entries'
import { stepsActivities } from '@/lib/db/schema/steps-activities'
import { areas } from '@/lib/db/schema/areas'

export async function getTimeByArea(period: ReportPeriod) {
  const userId = await getAuthenticatedUserId()
  const { from, to } = getPeriodRange(period)
  const rows = await db
    .select({
      areaId: stepsActivities.areaId,
      areaName: areas.name,
      totalSeconds: sql<number>`COALESCE(SUM(${timeEntries.durationSeconds}), 0)`,
    })
    .from(timeEntries)
    .innerJoin(stepsActivities, eq(timeEntries.stepActivityId, stepsActivities.id))
    .leftJoin(areas, eq(stepsActivities.areaId, areas.id))
    .where(and(
      eq(timeEntries.userId, userId),
      gte(timeEntries.createdAt, from),
      lte(timeEntries.createdAt, to),
      isNotNull(timeEntries.durationSeconds)
    ))
    .groupBy(stepsActivities.areaId, areas.name)
    .orderBy(desc(sql`SUM(${timeEntries.durationSeconds})`))
  return rows.map(r => ({ areaId: r.areaId ?? 'none', areaName: r.areaName ?? 'Sin área', totalSeconds: Number(r.totalSeconds) }))
}
```
