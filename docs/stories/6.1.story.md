## Story

**As a** usuario que quiere capturar una idea, tarea o evento sin interrumpir su flujo de trabajo,
**I want** una forma rápida de escribir texto libre en el Inbox y verlo inmediatamente en una lista,
**so that** no pierdo ninguna captura y puedo procesarlas después en orden cronológico.

## Acceptance Criteria

### AC1 — Página `/inbox` existe con campo de captura

**Given** el usuario navega a `/inbox`,
**When** la página carga,
**Then:**
- La página renderiza un `<textarea>` o `<input>` con placeholder "Captura cualquier idea, tarea o evento…"
- Un botón "Agregar" (o Enter) está visible y habilitado
- La página muestra la lista de items de inbox del usuario en orden cronológico inverso (más reciente primero)
- La página es un Server Component que obtiene los items vía `getInboxItems(userId)`

### AC2 — Captura de texto libre crea item `pending`

**Given** el usuario escribe texto en el campo de captura,
**When** hace clic en "Agregar" o presiona Enter (con Shift+Enter para nueva línea),
**Then:**
- Se llama el Server Action `createInboxItem(rawText, userId)` que inserta en `inbox_items`:
  - `userId` = usuario autenticado
  - `rawText` = texto ingresado (trimmed, sin trailing spaces)
  - `status` = `'pending'`
  - `createdAt` = timestamp actual
- El campo de captura se limpia después de la inserción exitosa
- `revalidatePath('/inbox')` se llama — el nuevo item aparece al tope de la lista
- Si `rawText` está vacío (solo spaces): el Server Action retorna error sin insertar

### AC3 — Vista lista muestra items con estado visual

**Given** el usuario tiene items en el inbox,
**When** la página `/inbox` carga,
**Then:**
- Cada item muestra: texto (`raw_text`), timestamp relativo ("hace 5 minutos", "ayer"), y badge de estado (`pending`, `processed`, `manual`, `discarded`)
- Items con status `pending` tienen badge amarillo/naranja
- Items con status `processed` tienen badge verde con tachado visual en el texto
- Items con status `manual` tienen badge azul
- Items con status `discarded` tienen badge gris, visualmente atenuados
- La lista soporta scroll infinito o paginación (mínimo 20 items por página)

### AC4 — Acción "Descartar" marca item como `discarded`

**Given** el usuario ve un item `pending` o `manual` en la lista,
**When** hace clic en el botón/ícono "Descartar" (×) del item,
**Then:**
- Se llama el Server Action `discardInboxItem(itemId, userId)` que actualiza `status` = `'discarded'`
- El item se atenúa visualmente o desaparece de la vista (si hay filtro activo)
- `revalidatePath('/inbox')` se llama

### AC5 — Filtro de vista por estado

**Given** la lista tiene items de varios estados,
**When** el usuario selecciona un filtro (Todos | Pendientes | Procesados | Descartados),
**Then:**
- La lista muestra solo items del estado seleccionado
- El filtro "Todos" (default) muestra todos excepto `discarded` (que requiere selección explícita)
- El filtro mantiene su estado durante la sesión (no necesita persistirse en DB)

### AC6 — Conteo de items pendientes visible en sidebar

**Given** el usuario tiene items con status `pending` en el inbox,
**When** ve el sidebar de navegación,
**Then:**
- El ícono/link de Inbox en el sidebar muestra un badge numérico con el conteo de items `pending`
- Si no hay items pendientes: el badge no aparece (no muestra "0")
- El conteo se actualiza al navegar (no requiere WebSocket — solo en navegación)

### AC7 — Tests unitarios para Server Actions

**Given** los Server Actions de Inbox,
**Then** deben existir tests unitarios en `tests/unit/inbox/inbox-actions.test.ts` que verifiquen:
- `createInboxItem`: texto válido → inserta con status `pending`
- `createInboxItem`: texto vacío → retorna error sin insertar
- `discardInboxItem`: item existente → actualiza status a `discarded`
- `getInboxItems`: retorna items del usuario ordenados por `createdAt` DESC

### Server Actions Pattern

[Source: docs/architecture/coding-standards.md#12.3]

```typescript
// actions/inbox.ts — patrón estándar del proyecto
'use server'

import { revalidatePath } from 'next/cache'
import { db } from '@/lib/db/client'
import { inboxItems } from '@/lib/db/schema/inbox-items'
import { createServerClient } from '@/lib/supabase/server'
import { eq, and, desc } from 'drizzle-orm'

function assertDatabaseUrl() {
  if (!process.env.DATABASE_URL) throw new Error('DATABASE_URL is not set')
}

export async function createInboxItem(rawText: string): Promise<{ success: boolean; error?: string }> {
  const trimmed = rawText.trim()
  if (!trimmed) return { success: false, error: 'El texto no puede estar vacío' }
  try {
    assertDatabaseUrl()
    const supabase = await createServerClient()
    const { data: { user } } = await supabase.auth.getUser()
    if (!user) return { success: false, error: 'No autenticado' }
    await db.insert(inboxItems).values({ userId: user.id, rawText: trimmed, status: 'pending' })
    revalidatePath('/inbox')
    return { success: true }
  } catch (err) {
    console.error('[createInboxItem] failed:', err)
    return { success: false, error: 'Error al guardar el item' }
  }
}

export async function discardInboxItem(itemId: string): Promise<{ success: boolean; error?: string }> { ... }

export async function getInboxItems(userId: string, status?: string): Promise<InboxItem[]> { ... }
```

### Queries DB Pattern

[Source: docs/architecture/coding-standards.md#12.3 + source-tree.md]

```typescript
// lib/db/queries/inbox.ts
import { db } from '@/lib/db/client'
import { inboxItems } from '@/lib/db/schema/inbox-items'
import { eq, and, desc } from 'drizzle-orm'
import type { InboxItem } from '@/lib/db/schema/inbox-items'

export async function getInboxItemsByUser(
  userId: string,
  statusFilter?: InboxItem['status']
): Promise<InboxItem[]> {
  const conditions = [eq(inboxItems.userId, userId)]
  if (statusFilter) conditions.push(eq(inboxItems.status, statusFilter))
  return db
    .select()
    .from(inboxItems)
    .where(and(...conditions))
    .orderBy(desc(inboxItems.createdAt))
    .limit(50)
}

export async function getPendingInboxCount(userId: string): Promise<number> {
  const result = await db
    .select({ count: sql<number>`count(*)::int` })
    .from(inboxItems)
    .where(and(eq(inboxItems.userId, userId), eq(inboxItems.status, 'pending')))
  return result[0]?.count ?? 0
}
```
