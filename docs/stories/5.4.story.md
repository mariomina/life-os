## Story

**As a** usuario que gestiona su mes,
**I want** ver en la Vista Mes mis actividades reales del mes completo con indicadores por día,
**so that** pueda tener visibilidad de mi carga mensual de un vistazo sin navegar día por día.

## Acceptance Criteria

### AC1 — Query `getActivitiesForMonth`

**Given** un usuario autenticado y una fecha cualquiera del mes,
**When** se llama `getActivitiesForMonth(userId, date)`,
**Then:**
- Devuelve todas las `steps_activities` del usuario donde `scheduledAt` cae dentro del mes calendario (día 1 00:00:00Z – último día 23:59:59.999Z) que contiene `date`
- Usa UTC arithmetic directo (mismo patrón que `getActivitiesForWeek` — sin `date-fns` local helpers) para calcular los límites: primer día del mes UTC, último día del mes UTC
- Incluye `areaName` (LEFT JOIN `areas`) y `habitTitle` (LEFT JOIN `habits`) — `null` si no aplica
- Incluye `areaColor` derivado de `areas.maslowLevel` usando `maslowLevelToColor()` (reutiliza de `calendar.ts`)
- Ordena por `scheduledAt ASC`
- Está ubicada en `lib/db/queries/calendar.ts` (junto a `getActivitiesForDay` y `getActivitiesForWeek`)
- Comparte la misma interfaz `ActivityForCalendar` — sin nuevo tipo

### AC2 — `page.tsx` carga actividades del mes actual

**Given** el calendario carga la Vista Mes (cuando el usuario la selecciona),
**When** el Server Component renderiza,
**Then:**
- `app/(app)/calendar/page.tsx` llama `getActivitiesForMonth(user.id, new Date())` para el mes actual
- Convierte el resultado a `ICalendarEvent[]` usando el mismo mapping inline de Stories 5.2/5.3 (sin nueva función)
- Reemplaza la llamada a `getActivitiesForWeek` — el Server Component ahora provee el mes completo al `CalendarClient`
- Si la query falla, renderiza el calendario con `events=[]` sin romper la página (error silencioso logueado)
- La Vista Semana sigue funcionando correctamente con los eventos del mes (WeekView filtra por semana internamente usando `getEventsForDay`)
- La Vista Día sigue funcionando correctamente (DayView filtra `currentDayEvents` internamente)

### AC3 — MonthView muestra datos reales con indicadores por día

**Given** el calendario muestra la Vista Mes,
**When** hay actividades en `steps_activities` para ese mes,
**Then:**
- Los eventos aparecen en las celdas correctas según su `scheduledAt` local (timezone local del navegador)
- Cada evento muestra su color de área (derivado de `maslowLevel`) — chips coloreados en la celda del día
- Se muestran máximo 3 eventos por celda; si hay más, aparece "+N más" (comportamiento ya implementado — sin regresión)
- Los días del mes actual con al menos 1 evento muestran un punto indicador (dot) bajo el número de día
- Al navegar a meses anteriores/siguientes, el `CalendarClient` muestra los eventos del `events` array ya provisto (sin nueva query — se re-fetch solo si el usuario recarga la página)

## Dev Notes

### Patrón de query (referencia: `getActivitiesForWeek` en `lib/db/queries/calendar.ts`)

```typescript
// lib/db/queries/calendar.ts — agregar debajo de getActivitiesForWeek
export async function getActivitiesForMonth(
  userId: string,
  date: Date
): Promise<ActivityForCalendar[]> {
  assertDatabaseUrl()

  // Compute month boundaries in UTC.
  // Month: UTC first day 00:00:00.000Z – UTC last day 23:59:59.999Z
  // Using Date.UTC() directly — avoid date-fns local timezone drift (gotcha de Story 5.3)
  const year = date.getUTCFullYear()
  const month = date.getUTCMonth() // 0-indexed

  const rangeStart = new Date(Date.UTC(year, month, 1, 0, 0, 0, 0))
  // Last day: Date.UTC(year, month+1, 0) = last day of `month`
  const lastDay = new Date(Date.UTC(year, month + 1, 0)).getUTCDate()
  const rangeEnd = new Date(Date.UTC(year, month, lastDay, 23, 59, 59, 999))

  // Query idéntica a getActivitiesForWeek — solo cambian los límites
}
```

**IMPORTANTE:** `Date.UTC(year, month + 1, 0)` retorna el último día del mes — técnica estándar JS. No usar `endOfMonth` de `date-fns` (opera en hora local del servidor).

```typescript
// Agregar en tests/unit/calendar/calendar-queries.test.ts

/** Mimics the month-range filter applied in getActivitiesForMonth (UTC) */
function isInMonth(scheduledAt: Date, date: Date): boolean {
  const year = date.getUTCFullYear()
  const month = date.getUTCMonth()
  const rangeStart = new Date(Date.UTC(year, month, 1, 0, 0, 0, 0))
  const lastDay = new Date(Date.UTC(year, month + 1, 0)).getUTCDate()
  const rangeEnd = new Date(Date.UTC(year, month, lastDay, 23, 59, 59, 999))
  return scheduledAt >= rangeStart && scheduledAt <= rangeEnd
}

// TARGET_MONTH_DATE = new Date('2024-01-15T00:00:00Z') → mes enero 2024
// Casos: Jan 1 00:00:00Z included, Jan 31 23:59:59.999Z included,
//        Dec 31 23:59:59.999Z excluded, Feb 1 00:00:00Z excluded
// Caso extra: febrero 2024 (29 días — año bisiesto) → Feb 29 23:59:59.999Z included
```
