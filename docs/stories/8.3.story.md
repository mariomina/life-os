## Story

**As a** usuario con al menos 14 días de datos (time entries, checkins, hábitos),
**I want** que el sistema calcule automáticamente correlaciones estadísticas entre mis métricas,
**so that** pueda descubrir patrones invisibles entre mis actividades y la salud de mis áreas.

## Acceptance Criteria

### AC1 — Función pura `computePearson`
**Given** dos arrays numéricos de igual longitud (series temporales de métricas),
**When** se llama a `computePearson(seriesA, seriesB)`,
**Then** retorna el coeficiente de Pearson (−1 a 1) o `null` si hay menos de 3 puntos o varianza cero.

### AC2 — Función pura `computeSpearman`
**Given** dos arrays numéricos de igual longitud,
**When** se llama a `computeSpearman(seriesA, seriesB)`,
**Then** retorna el coeficiente de Spearman (rank correlation) o `null` si hay menos de 3 puntos.

### AC3 — Función pura `classifyCorrelation`
**Given** un coeficiente de correlación y el número de días de datos,
**When** se llama a `classifyCorrelation(coefficient, daysOfData)`,
**Then** retorna `'positive' | 'negative' | 'neutral'` según umbrales:
- `positive`: coefficient ≥ 0.3
- `negative`: coefficient ≤ −0.3
- `neutral`: |coefficient| < 0.3
Y el tier: `'gathering'` (< 7 días), `'provisional'` (7-13 días), `'full'` (≥ 14 días).

### AC4 — Función pura `buildCorrelationPairs`
**Given** una lista de métricas de series temporales por área y por hábito,
**When** se llama a `buildCorrelationPairs(metrics)`,
**Then** retorna todos los pares a correlacionar (área vs. área, área vs. hábito, etc.).

### AC5 — Server Action `runCorrelationEngine()`
**Given** el usuario autenticado con datos suficientes,
**When** se llama a `runCorrelationEngine()`,
**Then**:
1. Lee los últimos 90 días de datos (time_entries por área, completions de hábitos por día, area_scores diarios)
2. Construye series temporales diarias para cada métrica
3. Calcula Pearson para todos los pares
4. Marca las correlaciones previas como `isActive=false`
5. Inserta las nuevas correlaciones en la tabla `correlations` con tier, type y confidence
6. Retorna `{ success: true, correlationsComputed: number }`

### AC6 — Tier mínimo de 14 días para correlaciones `full`
**Given** el motor ejecutando con datos,
**When** hay < 14 días de datos para un par de métricas,
**Then** la correlación se guarda con `tier='gathering'` o `tier='provisional'` y sin `correlationValue` (null).
Solo pares con ≥ 14 días obtienen `tier='full'` con coeficiente calculado.

### AC7 — Server Action `getActiveCorrelations()`
**Given** el usuario autenticado,
**When** se llama a `getActiveCorrelations()`,
**Then** retorna las correlaciones `isActive=true` del último run, ordenadas por |correlationValue| desc.

### AC8 — Tests ≥ anteriores + nuevos de Story 8.3
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan (funciones puras testeadas, no se testea la Edge Function directamente).

## Risks

- **Pocas métricas con 14+ días:** En usuarios nuevos, casi todo será `gathering`. El sistema debe comunicar esto claramente (no mostrar coeficientes falsos).
- **Varianza cero:** Si una métrica es constante (ej: área siempre a 0h), Pearson es indefinido — retornar null.
- **N×M pares explosivos:** Con 8 áreas + 10 hábitos = 18 métricas → 153 pares. Aceptable para MVP.

## Dev Notes

### Algoritmos puros en `features/correlations/engine.ts`

```typescript
// Pearson correlation coefficient
export function computePearson(a: number[], b: number[]): number | null {
  if (a.length !== b.length || a.length < 3) return null
  const n = a.length
  const meanA = a.reduce((s, x) => s + x, 0) / n
  const meanB = b.reduce((s, x) => s + x, 0) / n
  const num = a.reduce((s, x, i) => s + (x - meanA) * (b[i] - meanB), 0)
  const denA = Math.sqrt(a.reduce((s, x) => s + (x - meanA) ** 2, 0))
  const denB = Math.sqrt(b.reduce((s, x) => s + (x - meanB) ** 2, 0))
  if (denA === 0 || denB === 0) return null
  return num / (denA * denB)
}

// Spearman: rank the arrays, then apply Pearson on ranks
export function computeSpearman(a: number[], b: number[]): number | null {
  if (a.length !== b.length || a.length < 3) return null
  const rank = (arr: number[]) => {
    const sorted = [...arr].sort((x, y) => x - y)
    return arr.map(v => sorted.indexOf(v) + 1)
  }
  return computePearson(rank(a), rank(b))
}

export function classifyCorrelation(
  coefficient: number | null,
  daysOfData: number
): { type: 'positive' | 'negative' | 'neutral'; tier: 'gathering' | 'provisional' | 'full' } {
  const tier = daysOfData < 7 ? 'gathering' : daysOfData < 14 ? 'provisional' : 'full'
  if (coefficient === null || tier !== 'full') return { type: 'neutral', tier }
  if (coefficient >= 0.3) return { type: 'positive', tier }
  if (coefficient <= -0.3) return { type: 'negative', tier }
  return { type: 'neutral', tier }
}
```

### Formato de series temporales

Para construir series: crear array de 90 slots (uno por día), cada slot es 0 si no hay dato. Esto garantiza arrays de igual longitud para el cálculo de correlación.
