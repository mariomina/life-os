## Story

**As a** usuario con proyectos creados,
**I want** poder crear y editar workflows visualmente con un canvas interactivo,
**so that** pueda organizar mis Tasks y Steps con executor_type visible (human/ai/mixed) y dependencias en forma de grafo, sin necesidad de escribir código o YAML.

## Dependencies

- **Requires:** Story 4.1 — Done ✅ (proyectos existentes como entidad padre del workflow)
- **Requires:** Story 1.4 — Done ✅ (tablas `workflows`, `tasks`, `steps_activities` en schema Drizzle)
- **Blocks:** Story 4.3 (Templates instancian su canvas en la tabla `workflows.canvas_data`)
- **Blocks:** Story 4.4 (Squads se asignan a workflows creados aquí)
- **Blocks:** Story 4.5 (Steps tipo `ai`/`human` se procesan desde los steps creados aquí)

## Scope

### IN

- **Página `/projects/[id]/workflow`:** canvas React Flow para el primer workflow del proyecto
- **CRUD Workflow:** crear workflow al acceder a la página si no existe (uno por proyecto en MVP)
- **Nodos Task:** rectángulos azul-gris, arrastrable, con título editable inline
- **Nodos Step:** círculos coloreados por `executor_type`:
  - `human` → azul (`#3b82f6`)
  - `ai` → púrpura (`#8b5cf6`)
  - `mixed` → degradado azul→púrpura
- **Edges dirigidos:** flechas entre Steps (dependencias de secuencia)
- **Panel lateral (sidebar):** al hacer click en nodo muestra: título, descripción, executor_type (selector), ai_agent (input visible si ai/mixed), verification_criteria (textarea), status (badge)
- **Persistencia:** `workflows.canvas_data` (JSONB) guarda `{ nodes, edges }` al salvar; `tasks` y `steps_activities` se sincronizan desde el canvas
- **Botón "Guardar":** persiste canvas_data + sincroniza tasks/steps en DB vía Server Action
- **Toolbar mínimo:** botón "Add Task", botón "Add Step", botón "Guardar", botón "Auto-layout"
- **Auto-layout básico:** distribución left-to-right con dagre o ELK (librería separada)
- **Link desde `/projects/[id]`:** botón "Ver Workflow" en la página de detalle del proyecto
- **Server Actions:** `createWorkflow`, `getWorkflowByProject`, `saveWorkflowCanvas`
- **Query layer:** `getWorkflowWithTasks`, `getWorkflowCanvas` en `lib/db/queries/workflows.ts`
- **Tests unitarios:** lógica pura de transformación canvas↔DB (nodos→tasks, nodos→steps)

### OUT

- Mini-map (Phase 2)
- Zoom controls avanzados (Phase 2)
- Exportar como AIOS YAML (Phase 2)
- Múltiples workflows por proyecto (MVP = 1 por proyecto)
- Drag desde template gallery al canvas (Story 4.3)
- Squad visual en nodos AI (Story 4.4)
- Ejecución automática de steps (Story 4.5)
- Collaboration real-time (Phase 2)

## Acceptance Criteria

**AC1 — Acceso al canvas:**
Given el usuario está en `/projects/[id]`,
When hace click en "Ver Workflow",
Then navega a `/projects/[id]/workflow` y ve el canvas React Flow vacío (o con nodos si ya existían).

**AC2 — Crear Task:**
Given el usuario está en el canvas,
When hace click en "Add Task",
Then aparece un nodo Task (rectángulo) en el canvas con título editable "Nueva Task".

**AC3 — Crear Step:**
Given el usuario está en el canvas con al menos una Task,
When hace click en "Add Step",
Then aparece un nodo Step (círculo, color azul por defecto = human) en el canvas.

**AC4 — Colores por executor_type:**
Given un Step existe en el canvas,
When el usuario cambia `executor_type` a `ai` en el panel lateral,
Then el nodo del Step cambia a color púrpura (#8b5cf6).
When cambia a `mixed`,
Then el nodo muestra degradado azul→púrpura.

**AC5 — Conexiones entre Steps:**
Given dos nodos Step existen en el canvas,
When el usuario arrastra desde el handle de salida de un Step al handle de entrada de otro,
Then aparece una flecha dirigida entre ambos.

**AC6 — Panel lateral:**
Given el usuario hace click en un nodo (Task o Step),
When el panel lateral se abre,
Then muestra: título (editable), descripción (editable), executor_type (selector), ai_agent (input, visible solo si ai o mixed), verification_criteria (textarea), status (badge no editable).

**AC7 — Persistencia del canvas:**
Given el usuario ha añadido nodos y edges,
When hace click en "Guardar",
Then `workflows.canvas_data` se actualiza con `{ nodes, edges }`, y las tablas `tasks` y `steps_activities` se sincronizan con los nodos del canvas.

**AC8 — Reload del canvas:**
Given el usuario guardó un canvas y recarga la página,
When `/projects/[id]/workflow` carga,
Then el canvas restaura exactamente los mismos nodos, edges y posiciones.

**AC9 — Workflow autocreado:**
Given el usuario accede a `/projects/[id]/workflow` y el proyecto no tiene workflow,
Then el sistema crea automáticamente un workflow vacío (`status='active'`, `squadType='none'`) vinculado al proyecto.

**AC10 — Link desde detalle:**
Given el usuario está en `/projects/[id]`,
When ve la sección de Workflow,
Then hay un botón "Ver Workflow" que lo lleva a `/projects/[id]/workflow`.

## Dev Notes

### Schema existente (no requiere migración)

```typescript
// workflows: id, projectId, userId, title, description, templateId, squadType, status, canvasData, createdAt, updatedAt
// tasks: id, workflowId, userId, title, description, order, status
// steps_activities: id, taskId, userId, areaId, title, description, executorType, planned, aiAgent, verificationCriteria, status, order
```

`canvasData` es `jsonb` — shape: `{ nodes: FlowNode[], edges: FlowEdge[] }`

### Instalación de @xyflow/react

```bash
npm install @xyflow/react
```

Versión: v12 (MIT). Documentación: `https://reactflow.dev/`

**Import del CSS obligatorio** (en el componente o layout del canvas):
```typescript
import '@xyflow/react/dist/style.css'
```

### Tipos de nodo personalizados

```typescript
// TaskNode — rectángulo
type TaskNodeData = { label: string; status: Task['status'] }

// StepNode — círculo
type StepNodeData = {
  label: string
  executorType: 'human' | 'ai' | 'mixed'
  aiAgent?: string | null
  verificationCriteria?: string | null
  status: StepActivity['status']
}
```

### Colores por executor_type

```typescript
const EXECUTOR_COLORS = {
  human: '#3b82f6',   // blue-500
  ai: '#8b5cf6',       // violet-500
  mixed: 'linear-gradient(135deg, #3b82f6, #8b5cf6)',
}
```

### Auto-layout con dagre

```bash
npm install dagre @types/dagre
```

Layout left-to-right, spacing: `rankSep=80`, `nodeSep=60`.

### Patrón de Server Actions (seguir actions/projects.ts)

```typescript
// actions/workflows.ts
'use server'
export async function createWorkflow(projectId: string, title: string): Promise<ActionResult>
export async function getWorkflowByProject(projectId: string): Promise<Workflow | null>
export async function saveWorkflowCanvas(
  workflowId: string,
  canvasData: { nodes: FlowNode[]; edges: FlowEdge[] },
  tasks: TaskSyncData[],
  steps: StepSyncData[]
): Promise<ActionResult>
```

### Sincronización canvas → DB

Al guardar:
1. Leer nodos tipo `task` → upsert en tabla `tasks` (workflowId, title, order)
2. Leer nodos tipo `step` → upsert en tabla `steps_activities` (taskId según edge padre, title, executorType, aiAgent, verificationCriteria, order)
3. Guardar `canvas_data` en `workflows`

Los IDs de los nodos React Flow coinciden con los IDs de DB (UUID) para facilitar la sincronización.

### Ruta de la página

- Canvas: `app/(app)/projects/[id]/workflow/page.tsx` (Client Component — React Flow requiere browser)
- Componentes: `components/workflow/WorkflowCanvas.tsx`, `components/workflow/NodeSidebar.tsx`, `components/workflow/TaskNode.tsx`, `components/workflow/StepNode.tsx`

### React Flow requiere height explícita

El contenedor del canvas DEBE tener altura explícita:
```tsx
<div style={{ width: '100%', height: 'calc(100vh - 200px)' }}>
  <ReactFlow ... />
</div>
```

### Client Component obligatorio

React Flow usa `window` y `ResizeObserver` — debe ser Client Component con `'use client'`.
El Server Component padre (`page.tsx`) carga el workflow desde DB y lo pasa como prop al canvas.
