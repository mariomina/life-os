## Story

**As a** usuario con hábitos, activities y OKRs registrados,
**I want** ver métricas clave de mi sistema (consistencia de hábitos, tasa de compromiso del calendario, progreso de OKRs y tendencia de salud de áreas),
**so that** tenga un dashboard de métricas para el Weekly Review y pueda detectar desviaciones de mi plan.

## Acceptance Criteria

### AC1 — Función pura `computeHabitConsistency`
**Given** un array de hábitos con sus completions planificados y realizados en un período,
**When** se llama a `computeHabitConsistency(habits)`,
**Then** retorna `{ habitId, habitName, planned, completed, rate }[]` donde `rate = completed/planned` (0-1).

### AC2 — Server Action `getHabitConsistencyReport(period)`
**Given** el usuario autenticado y un período,
**When** se llama a `getHabitConsistencyReport(period)`,
**Then** retorna el array de consistencia por hábito usando activities con `habitId` no nulo en el período.

### AC3 — Función pura `computeCCR`
**Given** activities planificadas (`planned=true`) y completadas (`status='completed'`) en el período,
**When** se llama a `computeCCR(planned, completed)`,
**Then** retorna `{ planned: number, completed: number, rate: number }` donde `rate = completed/planned` (0-1, null si planned=0).

### AC4 — Server Action `getCalendarCommitmentRate(period)`
**Given** el usuario autenticado,
**When** se llama a `getCalendarCommitmentRate(period)`,
**Then** retorna CCR para el período: cuenta activities `planned=true` y `planned=true + status='completed'` en el rango.

### AC5 — Server Action `getOkrProgressReport()`
**Given** el usuario autenticado,
**When** se llama a `getOkrProgressReport()`,
**Then** retorna OKRs activos con sus KRs y el promedio de progreso:
`{ okrId, objective, krs: { krId, title, progress }[], avgProgress: number }[]`.

### AC6 — Función pura `computeAreaHealthTrend`
**Given** scores de un área en dos períodos (últimos 30 días vs. 30 días anteriores),
**When** se llama a `computeAreaHealthTrend(currentScore, previousScore)`,
**Then** retorna `'improving' | 'declining' | 'stable'` (mejora/deteriora si delta > 5%, estable si ≤ 5%).

### AC7 — Server Action `getAreaHealthTrends()`
**Given** el usuario autenticado,
**When** se llama a `getAreaHealthTrends()`,
**Then** retorna `{ areaId, areaName, currentScore, previousScore, trend }[]` usando los últimos registros en `area_scores`.

### AC8 — UI: sección "Métricas" en `/reports`
**Given** la página `/reports` (de Story 8.1),
**When** se renderiza la sección Métricas,
**Then** aparecen 4 tarjetas/secciones:
- Habit Consistency: % promedio + lista por hábito con barra
- CCR: porcentaje grande + texto "X de Y actividades completadas"
- OKR Progress: lista de OKRs con barra de progreso
- Area Health Trend: lista de áreas con indicador ↑/→/↓

### AC9 — Tests ≥ (574 + tests 8.1) + nuevos de Story 8.2
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan.

## Risks

- **`area_scores` puede estar vacío:** Si el usuario nunca hizo diagnóstico, retornar array vacío.
- **Hábitos sin activities:** Si `habitId` nunca se asignó a activities, la consistencia es 0% — mostrar clara.
- **OKRs sin KRs:** Retornar OKR con `krs: []` y `avgProgress: 0`.
- **`planned=0` en CCR:** Rate = null — mostrar "Sin actividades planificadas" en vez de 0%.

## Dev Notes

### Funciones puras en `features/reports/metrics.ts`

```typescript
export type HabitConsistencyItem = {
  habitId: string; habitName: string; planned: number; completed: number; rate: number
}

export function computeHabitConsistency(
  habits: { habitId: string; habitName: string; planned: number; completed: number }[]
): HabitConsistencyItem[] {
  return habits.map(h => ({ ...h, rate: h.planned > 0 ? h.completed / h.planned : 0 }))
}

export function computeCCR(planned: number, completed: number): { planned: number; completed: number; rate: number | null } {
  return { planned, completed, rate: planned > 0 ? completed / planned : null }
}

export function computeAreaHealthTrend(currentScore: number, previousScore: number): 'improving' | 'declining' | 'stable' {
  const delta = currentScore - previousScore
  if (delta > 5) return 'improving'
  if (delta < -5) return 'declining'
  return 'stable'
}
```
