## Story

**As a** usuario con correlaciones calculadas (Story 8.3),
**I want** que el sistema detecte patrones avanzados como bucles autodestructivos, puntos de apalancamiento y cuellos de botella,
**so that** pueda identificar qué actividades me benefician o perjudican desproporcionadamente y actuar en consecuencia.

## Acceptance Criteria

### AC1 — Función pura `detectDestructiveLoops`
**Given** un array de correlaciones activas,
**When** se llama a `detectDestructiveLoops(correlations)`,
**Then** retorna los pares `{ entityA, entityB, coefficient }` donde:
- Ambas entidades tienen correlación `negative` entre sí (coefficient ≤ −0.4)
- Y al menos una de ellas es del tipo `area` con correlación negativa sobre sí misma en el tiempo (degradación)
- Estos se etiquetan como `type: 'destructive_loop'`

### AC2 — Función pura `detectLeveragePoints`
**Given** un array de correlaciones activas,
**When** se llama a `detectLeveragePoints(correlations)`,
**Then** retorna las entidades (actividades/hábitos) con correlación positiva fuerte (≥ 0.6) sobre 2+ áreas distintas,
- Etiquetadas como `type: 'leverage_point'`
- Ordenadas por `avgPositiveImpact` desc.

### AC3 — Función pura `detectBottlenecks`
**Given** un array de correlaciones activas,
**When** se llama a `detectBottlenecks(correlations)`,
**Then** retorna las entidades donde:
- Tienen correlación negativa (≤ −0.3) con 2+ áreas
- Se etiquetan como `type: 'bottleneck'`

### AC4 — Server Action `runAdvancedPatternDetection()`
**Given** correlaciones activas (`isActive=true`) ya calculadas por 8.3,
**When** se llama a `runAdvancedPatternDetection()`,
**Then**:
1. Lee correlaciones activas del usuario
2. Ejecuta `detectDestructiveLoops + detectLeveragePoints + detectBottlenecks`
3. Actualiza el tipo (`type`) de las correlaciones existentes en DB que coincidan con los patrones detectados
4. Retorna `{ destructiveLoops, leveragePoints, bottlenecks }` con conteos

### AC5 — UI: sección "Patrones Detectados" en `/reports`
**Given** la página `/reports`,
**When** hay patrones detectados,
**Then** aparece sección "Patrones del Sistema" con:
- 🔴 Bucles autodestructivos: nombre de las entidades involucradas
- 🟢 Puntos de apalancamiento: actividad/hábito + impacto
- 🟡 Cuellos de botella: entidad bloqueante + áreas afectadas
Si no hay patrones: "No se detectaron patrones con los datos actuales (se necesitan ≥ 14 días)."

### AC6 — Tests ≥ anteriores + nuevos de Story 8.4
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan.

## Business Value

Implementa FR14 completo. La diferencia entre "ver métricas" (8.1-8.2) y "entender el sistema" (8.4). Un leverage point detectado puede multiplicar el impacto del usuario. Un destructive loop detectado puede romper un ciclo de sabotaje inconsciente.

## Risks

- **Pocos datos:** La mayoría de usuarios nuevos tendrán tier=gathering/provisional. El sistema debe comunicar "necesitas más datos" de forma clara sin frustrar.
- **False positives:** Correlación estadística no implica causalidad. El texto de UI debe decir "patrón detectado" no "causa demostrada".

## Dev Notes

### `features/correlations/patterns.ts`

```typescript
import type { Correlation } from '@/lib/db/schema/correlations'

export function detectDestructiveLoops(correlations: Correlation[]): Correlation[] {
  // Buscar pares negativos fuertes (≤ -0.4) con tier='full'
  return correlations.filter(c =>
    c.type === 'negative' &&
    c.tier === 'full' &&
    Number(c.correlationValue) <= -0.4
  ).map(c => ({ ...c, type: 'destructive_loop' as const }))
}

export function detectLeveragePoints(correlations: Correlation[]): Correlation[] {
  // Entidades con correlación positiva fuerte (≥ 0.6) en 2+ áreas
  const entityImpact = new Map<string, number>() // entityAId → count of strong positive correlations
  for (const c of correlations) {
    if (c.type === 'positive' && c.tier === 'full' && Number(c.correlationValue) >= 0.6 && c.entityAId) {
      entityImpact.set(c.entityAId, (entityImpact.get(c.entityAId) ?? 0) + 1)
    }
  }
  return correlations.filter(c =>
    c.entityAId && (entityImpact.get(c.entityAId) ?? 0) >= 2 && Number(c.correlationValue) >= 0.6
  ).map(c => ({ ...c, type: 'leverage_point' as const }))
}

export function detectBottlenecks(correlations: Correlation[]): Correlation[] {
  // Entidades con impacto negativo en 2+ áreas
  const entityNegativeCount = new Map<string, number>()
  for (const c of correlations) {
    if (c.type === 'negative' && c.tier === 'full' && c.entityAId) {
      entityNegativeCount.set(c.entityAId, (entityNegativeCount.get(c.entityAId) ?? 0) + 1)
    }
  }
  return correlations.filter(c =>
    c.entityAId && (entityNegativeCount.get(c.entityAId) ?? 0) >= 2
  ).map(c => ({ ...c, type: 'bottleneck' as const }))
}
```
