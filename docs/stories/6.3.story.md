## Story

**As a** usuario con items de Inbox ya procesados por IA (status='processed'),
**I want** ver la propuesta de slot sugerida y confirmarla con 1 clic,
**so that** se cree automáticamente una activity en mi calendario sin esfuerzo adicional.

## Acceptance Criteria

### AC1 — Server Action `confirmInboxProposal`
**Given** un inbox item con `status='processed'` y campos IA válidos (`aiSuggestedAreaId`, `aiSuggestedSlot`, `aiSuggestedTitle`, `aiSuggestedDurationMinutes`),
**When** el usuario llama a `confirmInboxProposal(itemId)`,
**Then** se inserta un nuevo registro en `steps_activities` con `title=aiSuggestedTitle`, `scheduledAt=aiSuggestedSlot`, `scheduledDurationMinutes=aiSuggestedDurationMinutes`, `areaId=aiSuggestedAreaId`, `okrId=aiSuggestedOkrId` (si existe), `executorType='human'`, `planned=true`, `status='pending'`,
Y el inbox item se actualiza con `stepActivityId=[nuevo_id]`,
Y la función retorna `{ success: true, stepActivityId }`.

### AC2 — Ownership guard
**Given** cualquier llamada a `confirmInboxProposal(itemId)`,
**When** el item no existe o pertenece a otro usuario,
**Then** retorna `{ success: false, error: 'Item no encontrado' }` sin crear ningún registro.

### AC3 — Guard de estado
**Given** un inbox item con `status='pending'`, `'processing'`, `'manual'`, o `'discarded'`,
**When** el usuario llama a `confirmInboxProposal(itemId)`,
**Then** retorna `{ success: false, error: 'El item no tiene propuesta IA lista' }` sin crear activity.

### AC4 — Guard de doble confirmación
**Given** un inbox item que ya tiene `stepActivityId` (ya fue confirmado),
**When** el usuario llama de nuevo a `confirmInboxProposal(itemId)`,
**Then** retorna `{ success: false, error: 'Este item ya fue confirmado' }` sin duplicar la activity.

### AC5 — UI: Tarjeta de propuesta para items 'processed'
**Given** un item con `status='processed'` en la lista del Inbox,
**When** la página se renderiza,
**Then** se muestra una tarjeta expandida con:
- Título propuesto (`aiSuggestedTitle`)
- Fecha/hora del slot (`aiSuggestedSlot` formateado en local)
- Duración estimada (`aiSuggestedDurationMinutes` en formato legible)
- Área sugerida (`aiSuggestedAreaId` — puede mostrar el ID si no se tiene el nombre)
- Botón **"✓ Confirmar"** que llama a `processInboxItem` → `confirmInboxProposal`
- Botón **"Procesar IA"** (si `status='pending'`) que llama a `processInboxItem`

### AC6 — UI: Feedback visual tras confirmación
**Given** el usuario hace clic en "✓ Confirmar",
**When** `confirmInboxProposal` retorna `{ success: true }`,
**Then** el item en la lista se actualiza visualmente (badge cambia o tarjeta se colapsa)
Y se muestra un mensaje de éxito breve: "Activity creada en el calendario".

### AC7 — Revalidación de rutas
**Given** una confirmación exitosa,
**When** `confirmInboxProposal` completa la inserción,
**Then** se llama a `revalidatePath('/inbox')` y `revalidatePath('/calendar')` para que ambas páginas reflejen el nuevo estado.

### AC8 — Tests ≥ 456 + nuevos de confirmación
**Given** la implementación completa de Story 6.3,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan (≥ 456 anteriores + nuevos tests de `confirmInboxProposal`).

## Business Value

El usuario puede convertir cualquier idea capturada en el Inbox en una activity real del calendario con un solo clic, cerrando el loop del flujo Inbox → Calendario que es el núcleo del MVP.

## Risks

- **stepActivityId ya null en schema** ✅ — campo existe en Drizzle (`inbox_items.stepActivityId`) desde el schema original, no requiere migración.
- **areaId null** — si `aiSuggestedAreaId` es null (no debería pasar después de Story 6.2), la inserción fallará por FK. Guard adicional recomendado.
- **Timezone display** — `aiSuggestedSlot` está en UTC; la UI debe formatearlo en hora local del usuario. Se usa `toLocaleString()` como MVP.

## Dev Notes

### Arquitectura del flujo completo

```
Inbox UI (pending item)
  → Botón "Procesar IA" → processInboxItem() [Story 6.2]
    → status='processed' + campos IA rellenados

Inbox UI (processed item — tarjeta de propuesta)
  → Botón "✓ Confirmar" → confirmInboxProposal(itemId) [Story 6.3]
    1. Valida ownership + status='processed' + no stepActivityId
    2. INSERT steps_activities {title, scheduledAt, ...}
    3. UPDATE inbox_items SET stepActivityId=newId
    4. revalidatePath('/inbox') + revalidatePath('/calendar')
    → { success: true, stepActivityId }
```

### Server Action `confirmInboxProposal`

```typescript
// actions/inbox.ts — añadir al final

export interface ConfirmInboxResult {
  success: boolean
  stepActivityId?: string
  error?: string
}

export async function confirmInboxProposal(itemId: string): Promise<ConfirmInboxResult>
```

Pasos internos:
1. `assertDatabaseUrl()` + `getAuthenticatedUserId()`
2. Query: `SELECT * FROM inbox_items WHERE id=itemId AND userId=userId LIMIT 1`
3. Guard: item not found → error
4. Guard: `item.status !== 'processed'` → error
5. Guard: `item.stepActivityId != null` → error (ya confirmado)
6. Guard: `!item.aiSuggestedAreaId` → error (sin área, no se puede insertar)
7. Insert en `steps_activities` con:
   - `userId`, `areaId: item.aiSuggestedAreaId`
   - `title: item.aiSuggestedTitle ?? item.rawText`
   - `scheduledAt: item.aiSuggestedSlot`
   - `scheduledDurationMinutes: item.aiSuggestedDurationMinutes`
   - `okrId: item.aiSuggestedOkrId ?? null`
   - `executorType: 'human'`, `planned: true`, `status: 'pending'`
8. Update `inbox_items SET stepActivityId=newActivity.id, updatedAt=now() WHERE id=itemId`
9. `revalidatePath('/inbox')`, `revalidatePath('/calendar')`
10. Return `{ success: true, stepActivityId: newActivity.id }`

### UI Changes en `InboxClient.tsx`

Modificaciones mínimas al componente existente:

1. **Import `processInboxItem` y `confirmInboxProposal`** de `@/actions/inbox`
2. **Handler `handleProcess(itemId)`** — llama a `processInboxItem`, actualiza item en estado optimista
3. **Handler `handleConfirm(itemId)`** — llama a `confirmInboxProposal`, actualiza item en estado
4. **`ProposalCard` sub-componente** — renderiza para items con `status='processed'`:
   - Muestra `aiSuggestedTitle`, slot formateado, duración, área
   - Botón "✓ Confirmar" con loading state
5. **Botón "Procesar IA"** en items con `status='pending'`
6. **Estado de éxito**: mensaje temporal tras confirmación

### Formato de slot para UI

```typescript
// Formato legible del slot (UTC → local)
function formatSlot(date: Date): string {
  return new Intl.DateTimeFormat('es', {
    weekday: 'short',
    day: 'numeric',
    month: 'short',
    hour: '2-digit',
    minute: '2-digit',
  }).format(date)
}

// Duración legible
function formatDuration(minutes: number): string {
  if (minutes < 60) return `${minutes} min`
  const h = Math.floor(minutes / 60)
  const m = minutes % 60
  return m > 0 ? `${h}h ${m}min` : `${h}h`
}
```

- `ai_suggested_slot` timestamptz
- `ai_suggested_title` text
- `ai_suggested_duration_minutes` int

- `user_id`, `area_id` (NOT NULL), `title` (NOT NULL), `status` (default 'pending')
