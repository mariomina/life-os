## Story

**As a** usuario activo con diagnóstico completado,
**I want** ver alertas automáticas cuando algún área de nivel 1-2 lleva >7 días sin actividad o cuando >80% de mi tiempo está concentrado en 1-2 áreas durante >14 días,
**so that** el sistema me avise proactivamente cuando mis D-Needs críticos están siendo descuidados y pueda tomar acción antes de entrar en crisis.

## Scope

**IN scope:**
- Lógica pura de detección de alertas en `features/maslow/alerts.ts`:
  - **Alerta tipo A — Abandono crítico:** área nivel 1-2 con `last_activity_at` null o >7 días
  - **Alerta tipo B — Desbalance sistémico:** >80% del tiempo total invertido concentrado en 1-2 áreas durante el período analizado (basado en `time_entries` acumulado)
- Banners de alerta en Home (`/`) y Vista Áreas (`/areas`) — componente `AlertBanner`
- Tipos de alerta: `critical` (nivel 1-2 sin actividad) y `warning` (desbalance)
- Tests unitarios: toda la lógica de `alerts.ts` (casos borde incluidos)
- TypeScript strict, sin `any`

**OUT of scope:**
- Notificaciones push o email
- Alerta tipo C (score <50% >14 días) — esa condición activa el bloqueo soft de Story 3.3
- Sugerencias de balanceo (FR19 — Weekly Review)
- Persistencia de alertas en DB (calculadas en runtime)
- Configuración de umbrales por el usuario

## Risks

| Riesgo | Probabilidad | Impacto | Mitigación |
|--------|-------------|---------|------------|
| `last_activity_at` null en áreas sin actividad registrada | Alta | Bajo | Tratar null como "nunca" → siempre alerta si nivel 1-2 |
| Tiempo total = 0 (usuario sin time tracking) | Alta | Bajo | Si total=0 no emitir alerta de desbalance (sin datos suficientes) |
| Falsos positivos en desbalance con pocos datos | Media | Bajo | Umbral mínimo: total >60 min para calcular desbalance |

## Acceptance Criteria

1. `getAlerts(areas, timeMap)` retorna array de alertas correctamente tipadas
2. Área nivel 1-2 con `last_activity_at` null → alerta `critical` "Abandono crítico"
3. Área nivel 1-2 con `last_activity_at` >7 días → alerta `critical` "Abandono crítico"
4. Área nivel 3-8 sin actividad → NO genera alerta crítica (solo niveles 1-2)
5. >80% tiempo concentrado en ≤2 áreas (y total >60 min) → alerta `warning` "Desbalance sistémico"
6. Si total de tiempo = 0 o <60 min → NO se emite alerta de desbalance
7. `AlertBanner` muestra alertas críticas en rojo y warnings en amarillo
8. Los banners aparecen en Home (`/`) y Vista Áreas (`/areas`)
9. Si no hay alertas, no se renderiza ningún banner
10. `npm run typecheck` PASS, `npm run lint` PASS (0 errores)
11. Tests unitarios PASS: todos los casos de `getAlerts`

### Lógica de abandono crítico

```typescript
const CRITICAL_INACTIVITY_DAYS = 7
const CRITICAL_LEVELS = [1, 2]

function daysSince(date: Date | string | null): number {
  if (!date) return Infinity
  return Math.floor((Date.now() - new Date(date).getTime()) / (1000 * 60 * 60 * 24))
}
```
