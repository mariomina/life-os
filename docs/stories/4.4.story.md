## Story

**As a** usuario que gestiona un workflow en su proyecto,
**I want** poder asignar un squad de agentes (Dev Squad, Research Squad, Personal Coach) a mi workflow,
**so that** los steps de tipo `ai` hereden automáticamente el agente del squad asignado y sepa qué equipo ejecuta el trabajo.

## Dependencies

- **Requires:** Story 4.3 — Done ✅ (Templates instanciados con `squad_type` copiado al workflow — ya persiste en `workflows.squad_type`)
- **Requires:** Story 4.2 — Done ✅ (Canvas React Flow + `saveWorkflowCanvas` + schema Tasks/Steps funcional)
- **Blocks:** Story 4.5 (Steps tipo `ai` → cola AIOS — necesita saber qué agente del squad ejecuta cada step)

## Scope

### IN

- **Selector de Squad en toolbar del canvas:** Dropdown/select con las 4 opciones (Dev Squad, Research Squad, Personal Coach, None) visible en el toolbar del canvas
- **Server Action `updateWorkflowSquad(workflowId, squadType)`:** Actualiza `workflows.squad_type` en DB
- **Helper `squadTypeToAgents(squadType)`:** Función pura que mapea squad → lista de agentes (ej. `dev` → `['@architect', '@dev', '@qa', '@devops']`)
- **Herencia automática al instanciar template:** Cuando se instancia un template, el `squad_type` del template se aplica al selector de squad del canvas
- **Indicador visual en nodos AI:** Los nodos Step con `executorType='ai'` muestran el agente del squad asignado (ej. "@dev") como badge/tooltip
- **Actualización en tiempo real:** Al cambiar el squad, los nodos AI del canvas actualizan su indicador visualmente sin recargar
- **Persistencia:** `workflows.squad_type` persiste al recargar la página; el selector muestra el squad actual
- **Query layer:** `updateWorkflowSquad()` en `lib/db/queries/workflows.ts` (o función en el archivo existente)
- **Tests unitarios:** `squadTypeToAgents()` mapeando los 4 squads + lógica de herencia en nodos AI

### OUT

- Squads personalizados por el usuario (solo los 4 predefinidos en MVP)
- Asignación de agentes individuales a steps específicos (Story 4.5 y posterior)
- Panel de configuración de squad con detalle de agentes (solo badge/tooltip en MVP)
- Notificaciones a agentes cuando son asignados
- Historial de cambios de squad
- Squad visible desde la vista de proyecto (solo en canvas en MVP)

### Mapeo de squads predefinidos

```typescript
// lib/workflow/squad-utils.ts (archivo nuevo)
export type SquadType = 'dev' | 'research' | 'coach' | 'none'

export const SQUAD_LABELS: Record<SquadType, string> = {
  dev: 'Dev Squad',
  research: 'Research Squad',
  coach: 'Personal Coach',
  none: 'Sin Squad',
}

export function squadTypeToAgents(squadType: SquadType): string[] {
  const map: Record<SquadType, string[]> = {
    dev: ['@architect', '@dev', '@qa', '@devops'],
    research: ['@analyst', '@pm'],
    coach: ['@analyst'],
    none: [],
  }
  return map[squadType] ?? []
}

// Primer agente del squad = agente principal asignado a steps AI
export function squadTypeToPrimaryAgent(squadType: SquadType): string | null {
  const agents = squadTypeToAgents(squadType)
  return agents[0] ?? null
}
```

### Lógica de herencia en nodos AI (tiempo real)

Cuando el usuario cambia el squad:
1. Estado `activeSquadType` se actualiza
2. `squadTypeToPrimaryAgent(activeSquadType)` devuelve el agente
3. Los nodos AI del canvas se actualizan con `setNodes()` para incluir `data.aiAgent = agente`
4. Se llama `updateWorkflowSquad()` para persistir en DB (optimistic update)

### StepNode — badge de agente

`StepNode.tsx` ya recibe `data.aiAgent?: string`. Cuando `executorType='ai'` y `aiAgent` no es null, mostrar el badge con el agente (ej. "@dev"). Si squad es `none`, mostrar el campo vacío o "Sin agente" con estilo atenuado.

### Patrones existentes a seguir

- `WorkflowCanvas.tsx` — patrón de toolbar con botones (`addTask`, `addStep`, `autoLayout`, `Usar Template`)
- `handleSelectTemplate()` — patrón de Server Action + actualización de estado local
- `saveWorkflowCanvas()` en `actions/workflows.ts` — patrón de verify ownership + update + revalidatePath
- `StepNode.tsx` — nodo existente que ya maneja `executorType` y `aiAgent`

### RLS

La query `UPDATE workflows SET squad_type=... WHERE id=... AND user_id=...` está protegida por `user_id` en el WHERE. El `createClient()` de Supabase en Server Actions usa el contexto de sesión del usuario.

### Riesgo: Optimistic Update

Si `updateWorkflowSquad()` falla después de que el estado local se actualiza, el canvas puede mostrar un squad incorrecto. El @dev debe implementar rollback del estado en caso de error:

```typescript
const prevSquad = activeSquadType
setActiveSquadType(newSquad)           // optimistic update
const result = await updateWorkflowSquad(workflowId, newSquad)
if (result.error) {
  setActiveSquadType(prevSquad)         // rollback
  setSaveError(result.error)
}
```
