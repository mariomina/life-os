## Story

**As a** usuario que ejecuta una actividad con timer activo,
**I want** ver el tiempo transcurrido actualizarse en tiempo real en la UI sin recargar la página,
**so that** tenga retroalimentación visual inmediata del tiempo que llevo invertido en la actividad actual.

- **Requires:** Story 5.8 — Done ✅ (time tracking start/stop/pause/resume operativo, `time_entries` con `is_active=true`)
- **Requires:** Story 1.2 — Done ✅ (Supabase Auth — client-side `supabase` disponible en Client Components)

## Acceptance Criteria

### AC1 — Reloj en vivo (elapsed time) en DayView

**Given** el usuario tiene un timer activo (`is_active=true`) para una actividad y está en la Vista Día,
**When** el timer está corriendo,
**Then:**
- El evento muestra un contador `⏱ Xh Ym Zs` que se actualiza cada segundo (o cada 5s aceptable)
- El tiempo mostrado = `timeTotals[activityId]` (acumulado de sesiones anteriores) + tiempo transcurrido desde `startedAt` de la sesión activa
- El contador se actualiza localmente con `setInterval` — no necesita round-trip al servidor cada tick
- El formato es: `0s`, `1m 30s`, `1h 5m` según magnitud del tiempo transcurrido
- Cuando el timer se detiene (Detener o Pausar), el contador deja de correr y muestra el tiempo final

### AC2 — Supabase Realtime — sincronización multi-tab / multi-dispositivo

**Given** el usuario abre la misma sesión en dos tabs del navegador,
**When** inicia un timer en el Tab A,
**Then:**
- El Tab B detecta el cambio en `time_entries` vía Supabase Realtime (canal Postgres Changes) en menos de 1 segundo
- El Tab B actualiza su UI para mostrar los botones "Detener" + "Pausar" para esa actividad
- El Tab B inicia su propio `setInterval` local para el reloj en vivo desde el `started_at` recibido

**Given** el usuario detiene el timer en el Tab A,
**When** el Server Action `stopTimer` actualiza `is_active=false` en `time_entries`,
**Then:**
- El Tab B recibe el evento Realtime y detiene el reloj en vivo (elimina el `setInterval`)
- El Tab B actualiza los botones de vuelta a "Iniciar"

### AC3 — Canal Realtime solo para el usuario autenticado

**Given** la suscripción Realtime en `CalendarClient`,
**When** se establece el canal,
**Then:**
- El filtro del canal es `user_id=eq.{userId}` — solo recibe eventos de las entradas propias del usuario
- El canal se suscribe a tabla `time_entries` en evento `*` (INSERT, UPDATE, DELETE)
- En el `useEffect` de cleanup, el canal se elimina con `supabase.removeChannel(channel)` al desmontar el componente

### AC4 — Inicialización correcta del timer activo en page load

**Given** el usuario carga (o recarga) la página `/calendar` con un timer ya activo,
**When** el Server Component fetches `getActiveTimersForActivities`,
**Then:**
- `initialActiveTimers` ya contiene el `entryId` activo (implementado en Story 5.8)
- `CalendarClient` recibe además `initialTimerStartedAt: Record<string, string>` — el `startedAt` ISO string de la sesión activa para calcular elapsed time correctamente desde page load
- El reloj en vivo comienza desde `Date.now() - (Date.now() - new Date(startedAt).getTime())` — es decir, muestra el tiempo real transcurrido desde que se inició el timer, no desde que se cargó la página

### AC5 — Supabase client en Client Component (instancia reutilizable)

**Given** la necesidad de Supabase Realtime en `CalendarClient` (Client Component),
**When** se implementa la suscripción,
**Then:**
- Se usa `createSupabaseBrowserClient()` (cliente browser, importado de `@/lib/supabase/client`) — NO el server client
- El cliente se crea UNA vez con `useMemo` o como constante fuera del componente — no se instancia en cada render
- No hay imports de `'@/lib/supabase/server'` en Client Components

### AC6 — Tests: lógica del reloj en vivo

### Supabase Realtime — Postgres Changes

Supabase Realtime permite suscribirse a cambios en tablas PostgreSQL:

```typescript
// Patrón de suscripción — [Source: docs/architecture/tech-stack.md]
import { createSupabaseBrowserClient } from '@/lib/supabase/client'

const supabase = createSupabaseBrowserClient()

const channel = supabase
  .channel('timer-realtime')
  .on(
    'postgres_changes',
    {
      event: '*',           // INSERT, UPDATE, DELETE
      schema: 'public',
      table: 'time_entries',
      filter: `user_id=eq.${userId}`,
    },
    (payload) => {
      // payload.eventType: 'INSERT' | 'UPDATE' | 'DELETE'
      // payload.new: new row data
      // payload.old: old row data (para UPDATE/DELETE)
      handleTimerChange(payload)
    }
  )
  .subscribe()

// Cleanup al desmontar
return () => { supabase.removeChannel(channel) }
```

**IMPORTANTE:** Supabase Realtime Postgres Changes requiere que RLS esté habilitado en la tabla `time_entries` con policies que permitan SELECT al usuario dueño. La tabla ya tiene RLS desde Story 1.4.

### Reloj en vivo — patrón de implementación

```typescript
// En DayView o en CalendarClient — useEffect con setInterval
useEffect(() => {
  if (activeTimers.size === 0) return // sin timers activos, no iniciar interval

  const interval = setInterval(() => {
    setElapsedTick(Date.now()) // fuerza re-render para recalcular elapsed
  }, 1000)

  return () => clearInterval(interval)
}, [activeTimers.size]) // re-subscribe cuando cambia el número de timers activos

// En el render del evento:
const timerState = activeTimers.get(evt.id)
const startedAt = timerStartedAt.get(evt.id) // Map<activityId, Date>
const elapsed = timerState && !timerState.isPaused && startedAt
  ? Math.floor((Date.now() - startedAt.getTime()) / 1000)
  : 0
const accumulated = timeTotals[evt.id] ?? 0
const totalDisplay = formatElapsed(elapsed + accumulated)
```

### formatElapsed — función pura (distinta de formatSeconds)

```typescript
// Formato específico para timer en vivo con segundos visibles
export function formatElapsed(totalSeconds: number): string {
  if (totalSeconds <= 0) return '0s'
  const h = Math.floor(totalSeconds / 3600)
  const m = Math.floor((totalSeconds % 3600) / 60)
  const s = totalSeconds % 60
  if (h > 0) return m > 0 ? `${h}h ${m}m` : `${h}h`
  if (m > 0) return `${m}m ${s}s`
  return `${s}s`
}

export function calcElapsedSeconds(startedAt: Date, now: Date): number {
  const elapsed = Math.floor((now.getTime() - startedAt.getTime()) / 1000)
  return Math.max(0, elapsed) // nunca negativo
}
```

### getActiveTimerStartTimes — nueva query en actions/timer.ts

```typescript
// Añadir a actions/timer.ts
export async function getActiveTimerStartTimes(
  userId: string,
  activityIds: string[]
): Promise<Record<string, string>> {
  if (activityIds.length === 0) return {}
  const rows = await db
    .select({ stepActivityId: timeEntries.stepActivityId, startedAt: timeEntries.startedAt })
    .from(timeEntries)
    .where(
      and(
        eq(timeEntries.userId, userId),
        inArray(timeEntries.stepActivityId, activityIds),
        eq(timeEntries.isActive, true)
      )
    )
  return Object.fromEntries(rows.map((r) => [r.stepActivityId, r.startedAt.toISOString()]))
}
```
