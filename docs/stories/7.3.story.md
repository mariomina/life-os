## Story

**As a** usuario que registra tiempo en actividades,
**I want** que el sistema detecte automáticamente cuando he invertido mucho tiempo en un tema recurrente y me sugiera registrarlo como habilidad,
**so that** no tenga que rastrear manualmente mis skills emergentes.

## Acceptance Criteria

### AC1 — Función pura `detectEmergingSkills`
**Given** un array de activities con sus títulos y los skillIds ya registrados del usuario,
**When** se llama a `detectEmergingSkills(activities, existingSkillNames)`,
**Then** retorna una lista de sugerencias `{ term: string, totalSeconds: number, activityCount: number }[]` para términos que:
- Aparecen en ≥ 5 activities distintas (por título/palabras clave)
- Acumulan ≥ 3600 segundos (1h) de tiempo total
- NO existen ya como skill del usuario (por nombre exacto, case-insensitive)

### AC2 — Solo retorna términos > umbral
**Given** términos frecuentes en actividades,
**When** algún término tiene < 5 ocurrencias o < 3600s,
**Then** NO aparece en las sugerencias (filtrado estricto por ambos criterios).

### AC3 — Server Action `suggestSkillFromActivities`
**Given** el usuario autenticado,
**When** se llama a `suggestSkillFromActivities()`,
**Then** se obtienen las activities del usuario de los últimos 90 días con sus time entries,
Se ejecuta `detectEmergingSkills`,
Y retorna el array de sugerencias (puede ser vacío).

### AC4 — Server Action `confirmEmergingSkill`
**Given** una sugerencia con `term` como nombre,
**When** el usuario confirma (`confirmEmergingSkill(term, level, areaId?)`),
**Then** se llama internamente a `createSkill` con `autoDetected=true`,
Y retorna `{ success: true, skillId }`.

### AC5 — Server Action `dismissEmergingSkill`
**Given** una sugerencia mostrada al usuario,
**When** el usuario la descarta (`dismissEmergingSkill(term)`),
**Then** se guarda en una lista de términos descartados (localStorage o DB) para no volver a sugerir.

### AC6 — UI: banner de sugerencias en `/skills`
**Given** `suggestSkillFromActivities` retorna sugerencias no descartadas,
**When** se renderiza `/skills`,
**Then** aparece una sección "Habilidades emergentes detectadas" con cards por sugerencia:
- Nombre del término detectado
- "X horas en Y actividades"
- Botón "✓ Registrar skill" (abre mini-form con nivel + área)
- Botón "Ignorar"

### AC7 — Skills auto-detectadas marcadas visualmente
**Given** una skill creada vía `confirmEmergingSkill` (`autoDetected=true`),
**When** aparece en la lista de /skills,
**Then** muestra un badge "Auto" o indicador visual que distingue skills detectadas de manuales.

### AC8 — Tests ≥ 546 + nuevos de Story 7.3
**Given** la implementación completa,
**When** se ejecuta `npm test`,
**Then** todos los tests pasan (≥ 546 anteriores + nuevos de Story 7.3).

## Business Value

Cierra FR7 completamente: el sistema detecta habilidades emergentes sin intervención manual. El usuario no necesita saber a priori qué habilidades tiene — el sistema las infiere desde los datos de ejecución. Primera implementación del motor de detección de patrones (precursor del E8 correlations engine).

## Risks

- **Tokenización naive:** El algoritmo simple divide por palabras y busca frecuencias. Puede sugerir términos demasiado genéricos ("meeting", "review") que no son skills reales. Mitigación: lista de stop words básica (meeting, call, check, review, daily, weekly) para filtrar términos no-skill.
- **Activities sin time entries:** Si una activity nunca tuvo timer, su `durationSeconds` es 0. El filtro de ≥ 3600s la excluye automáticamente.
- **Dismiss persistence:** Usar tabla `skill_dismissed_suggestions (userId, term, dismissedAt)` en DB o simplemente localStorage. Para MVP: localStorage con key `dismissed_skill_suggestions`.

## Dev Notes

### Función pura `detectEmergingSkills`

Archivo nuevo: `features/skills/detection.ts`

```typescript
export interface EmergingSkillSuggestion {
  term: string
  totalSeconds: number
  activityCount: number
}

const STOP_WORDS = new Set([
  'meeting', 'call', 'check', 'review', 'daily', 'weekly',
  'session', 'work', 'task', 'item', 'todo', 'misc',
])

const MIN_ACTIVITIES = 5
const MIN_SECONDS = 3600

export function detectEmergingSkills(
  activities: { title: string; totalSeconds: number }[],
  existingSkillNames: string[]
): EmergingSkillSuggestion[] {
  const existingLower = new Set(existingSkillNames.map(n => n.toLowerCase()))

  // Tokenize titles → word frequency map
  const termMap = new Map<string, { totalSeconds: number; activityCount: number }>()

  for (const activity of activities) {
    const words = activity.title
      .toLowerCase()
      .replace(/[^a-záéíóúñüa-z0-9\s]/gi, '')
      .split(/\s+/)
      .filter(w => w.length > 3 && !STOP_WORDS.has(w))

    for (const word of words) {
      const existing = termMap.get(word) ?? { totalSeconds: 0, activityCount: 0 }
      termMap.set(word, {
        totalSeconds: existing.totalSeconds + activity.totalSeconds,
        activityCount: existing.activityCount + 1,
      })
    }
  }

  return Array.from(termMap.entries())
    .filter(([term, data]) =>
      data.activityCount >= MIN_ACTIVITIES &&
      data.totalSeconds >= MIN_SECONDS &&
      !existingLower.has(term)
    )
    .map(([term, data]) => ({ term, ...data }))
    .sort((a, b) => b.totalSeconds - a.totalSeconds)
}
```
