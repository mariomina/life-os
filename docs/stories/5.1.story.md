### Patrón de integración (Client Component)

```tsx
// app/(app)/calendar/_components/CalendarClient.tsx
'use client'
import { Calendar } from '@lramos33/big-calendar'
import './calendar.css'  // overrides Tailwind v3→v4

interface CalendarClientProps {
  // stub events para AC5
  events?: Array<{ id: string; title: string; start: Date; end: Date }>
}

export function CalendarClient({ events = [] }: CalendarClientProps) {
  return (
    <Calendar
      events={events}
      defaultView="week"
      locale="es"
    />
  )
}
```

### Análisis Tailwind v3→v4 — clases problemáticas conocidas

big-calendar (y sus dependencias) pueden usar clases Tailwind v3 que no existen en v4:
- `bg-opacity-{n}` → en v4 usar `bg-{color}/{opacity}` o CSS custom
- `text-opacity-{n}` → ídem
- `ring-offset-*` → puede necesitar override
- Prefijo `dark:` en v3 puede comportarse diferente en v4

El enfoque correcto: instalar el paquete, ejecutar `npm run dev`, observar warnings en consola y errores visuales, luego crear los overrides mínimos en `calendar.css`.

## Testing

### Estrategia de tests para Story 5.1

Story 5.1 es de **integración de infraestructura** — el foco de tests es en:
1. Funciones puras de utilidad (`calendar-utils.ts`) → Vitest unitario
2. Verificación de que el build y typecheck pasan con el nuevo paquete

**No se requieren** tests de componente React para el wrapper de big-calendar en esta story — ese nivel de testing se añade en Stories 5.2+ cuando hay lógica de datos real.

### Ejemplos de tests unitarios (`calendar-utils.test.ts`)

```typescript
import { describe, it, expect } from 'vitest'
import { toCalendarEvent, getWeekRange, formatDateHeader } from '@/lib/calendar/calendar-utils'

describe('toCalendarEvent', () => {
  it('converts a steps_activity to big-calendar event format', () => {
    const activity = {
      id: 'abc',
      title: 'Meditación',
      scheduledAt: new Date('2024-01-15T07:00:00Z'),
      scheduledDurationMinutes: 30,
    }
    const event = toCalendarEvent(activity)
    expect(event.id).toBe('abc')
    expect(event.start).toBeInstanceOf(Date)
    expect(event.end.getTime() - event.start.getTime()).toBe(30 * 60 * 1000)
  })
})

describe('getWeekRange', () => {
  it('returns Monday-Sunday for a given date', () => {
    const { start, end } = getWeekRange(new Date('2024-01-15')) // Monday
    expect(start.getDay()).toBe(1) // Monday
    expect(end.getDay()).toBe(0)   // Sunday (or 7 depending on locale)
  })
})
```
