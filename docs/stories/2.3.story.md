## Story

**As a** usuario nuevo que eligió "Subir archivos" en el onboarding,
**I want** poder pegar o subir texto/archivos con mis resultados psicométricos (MBTI, Big Five, StrengthsFinder, etc.) para que la IA los analice y genere mi score inicial en las 8 áreas Maslow,
**so that** obtenga mi diagnóstico sin tener que responder el cuestionario manual, usando mis resultados psicológicos existentes como base.

## Dependencies

- **Requires:** Story 2.1 — Done ✅ (onboarding wizard, `/onboarding/upload` placeholder, `user_metadata.onboarding_method = 'upload'`)
- **Requires:** Story 2.2 — Done ✅ (`seedUserAreas`, `saveAreaScores`, `completeOnboarding` Server Actions)

## Scope

**IN scope:**
- Reemplazar el placeholder de `/onboarding/upload/page.tsx` con la UI real de upload/paste
- UI: textarea para pegar texto de resultados psicométricos (texto libre) + botón opcional para subir archivo .txt/.pdf
- Lectura del archivo en cliente → extracción de texto plano
- Envío del texto a un Server Action que llama al proveedor de IA (Claude via Anthropic SDK)
- Prompt de IA: analizar el texto psicométrico y devolver JSON con scores 0-100 para las 8 áreas Maslow
- Interfaz `ILLMProvider` — abstracción multi-proveedor (`lib/ai/providers/`)
- Implementar proveedor Claude (`lib/ai/providers/claude.ts`) como default
- Pantalla de revisión: mostrar scores propuestos por IA antes de confirmar (mismo formato que DiagnosisSummary de Story 2.2)
- Al confirmar: reutilizar `seedUserAreas` + `saveAreaScores` + `completeOnboarding` de Story 2.2
- Redirect a `/` (Home) tras confirmar
- TypeScript strict, sin `any`
- Tests unitarios: parser del output JSON de IA, validación de scores (0-100 range)

**OUT of scope:**
- Subir archivos a Supabase Storage (se lee en cliente, no se persiste el archivo)
- Múltiples proveedores de IA simultáneos (solo Claude en MVP)
- Fine-tuning o RAG del modelo
- Validación psicométrica del output de IA
- Soporte para PDFs con imágenes escaneadas (solo texto extraíble)

## Complexity

**Estimación:** M — 5 story points
- Razón: Integración con Anthropic SDK (nueva dependencia), interfaz ILLMProvider, prompt engineering, UI de upload, pantalla de revisión. Reutiliza actions de Story 2.2 para el guardado.

## Risks

| Riesgo | Probabilidad | Impacto | Mitigación |
|--------|-------------|---------|------------|
| API key de Anthropic no configurada en local/CI | Alta | Alto | Usar mock del proveedor en tests; leer key de `ANTHROPIC_API_KEY` env var |
| Output JSON de IA mal formateado | Media | Alto | Validar y sanitizar JSON antes de usar; fallback a scores por defecto |
| Archivo PDF sin texto extraíble | Media | Medio | Solo aceptar .txt en MVP; mostrar mensaje claro al usuario |
| Costo de API en CI/CD | Baja | Medio | Tests unitarios mockean el proveedor — sin llamadas reales |

## Acceptance Criteria

1. La ruta `/onboarding/upload` muestra la UI real (no el placeholder)
2. El usuario puede pegar texto en un textarea de mínimo 200 caracteres o subir un archivo .txt
3. Al enviar, se muestra un indicador de carga mientras la IA procesa
4. La IA devuelve scores 0-100 para las 8 áreas Maslow en formato JSON estructurado
5. Se muestra una pantalla de revisión con los scores propuestos (mismo formato que Story 2.2: barra por área + Life System Health Score global)
6. El usuario puede confirmar o volver atrás para re-intentar
7. Al confirmar: `seedUserAreas` + `saveAreaScores` + `completeOnboarding` se ejecutan en secuencia
8. Tras guardar, redirect a `/` y ya no hay loop al onboarding
9. Tests unitarios PASS: validación de scores IA (range 0-100, todas las áreas presentes)
10. `npm run typecheck` PASS, `npm run lint` PASS (0 errores)
