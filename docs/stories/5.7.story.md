## Story

**As a** usuario que planifica su tiempo,
**I want** crear y eliminar actividades directamente desde el calendario con un formulario modal,
**so that** pueda registrar nuevos compromisos o borrar actividades canceladas sin salir de la vista de calendario.

## Acceptance Criteria

### AC1 — Botón "Nueva Actividad" en toolbar del Calendario

**Given** el usuario está en cualquier vista del calendario,
**When** ve el toolbar superior,
**Then:**
- Existe un botón "+ Nueva Actividad" en el toolbar (lado derecho, junto al selector de vistas)
- Al hacer clic, abre un modal/dialog de creación de actividad
- El botón es visible en todas las vistas (Día, Semana, Mes, Año, Agenda)

### AC2 — Modal de creación con campos mínimos

**Given** el usuario abre el modal de nueva actividad,
**When** ve el formulario,
**Then:**
- El modal contiene los campos:
  - **Título** (text, requerido, max 100 chars)
  - **Fecha** (date input, requerido, por defecto = `currentDate`)
  - **Hora inicio** (time input, requerido, por defecto = próxima hora redonda)
  - **Duración** (select: 15m / 30m / 45m / 60m / 90m / 120m, por defecto 30m)
  - **Área** (select con las áreas del usuario, requerido)
- Botón "Crear" (submit) y botón "Cancelar"
- El modal es accesible (labels correctos, focus management, Escape cierra)

### AC3 — Server Action `createActivity` persiste en DB

**Given** el usuario completa el formulario y hace clic en "Crear",
**When** se procesa el submit,
**Then:**
- Se llama un Server Action `createActivity(formData)` que inserta en `steps_activities`:
  - `userId` = usuario autenticado (desde `supabase.auth.getUser()`)
  - `title` = título del formulario
  - `scheduledAt` = fecha + hora combinadas en timestamp con timezone
  - `scheduledDurationMinutes` = duración seleccionada en minutos
  - `areaId` = área seleccionada
  - `status` = `'pending'`
  - `planned` = `false` (actividad espontánea)
  - `executorType` = `'human'`
- Si la inserción es exitosa: el modal se cierra y la página se refresca (`revalidatePath('/calendar')`)
- Si falla: muestra mensaje de error dentro del modal (no cierra)

### AC4 — Botón "Eliminar" en eventos de DayView

**Given** el usuario está en la Vista Día y hace hover sobre un evento,
**When** el evento muestra sus action buttons,
**Then:**
- Aparece un botón "Eliminar" junto a los botones "Iniciar", "Check-in" y "Omitir"
- Al hacer clic en "Eliminar": solicita confirmación con `window.confirm('¿Eliminar esta actividad?')`
- Si confirma: llama Server Action `deleteActivity(activityId)` que elimina la fila de `steps_activities`
- Tras eliminar exitosamente: la página se refresca (`revalidatePath('/calendar')`)
- Botón visible solo para actividades espontáneas (`planned = false`) — las planeadas muestran solo "Omitir"

### AC5 — Query de áreas para el select del modal

**Given** el modal de nueva actividad necesita mostrar las áreas del usuario,
**When** el componente modal carga,
**Then:**
- Se usa un Server Action o una API route para obtener las áreas del usuario (`areas` table, filtradas por `userId`)
- El select muestra nombre del área y color visual (indicador de color por nivel Maslow)
- Si no hay áreas (usuario nuevo sin setup), muestra mensaje "Configura tus áreas primero"
- La lista se carga una sola vez al abrir el modal (sin polling)

### AC6 — Tests: Server Actions y validación

**Then:**

  2. Validación: título > 100 chars → error de validación
  3. Validación: sin área seleccionada → error de validación
  4. `scheduledAt` combinación correcta de fecha + hora (timezone-safe)
  5. `deleteActivity`: verifica que el ID es un UUID válido antes de llamar a DB

## Dev Notes

### Server Actions — patrón del proyecto

El proyecto ya usa Server Actions en `app/actions/`. Ver `app/actions/` para ejemplos de patrón.

```typescript
// app/actions/calendar.ts
'use server'

import { revalidatePath } from 'next/cache'
import { createSupabaseServerClient } from '@/lib/supabase/server'
import { db, assertDatabaseUrl } from '@/lib/db/client'
import { stepsActivities } from '@/lib/db/schema/steps-activities'
import { areas } from '@/lib/db/schema/areas'
import { eq } from 'drizzle-orm'

export async function createActivity(formData: FormData) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Unauthorized')

  const title = formData.get('title') as string
  const date = formData.get('date') as string      // 'YYYY-MM-DD'
  const time = formData.get('time') as string      // 'HH:mm'
  const duration = Number(formData.get('duration'))
  const areaId = formData.get('areaId') as string

  // Validate
  if (!title?.trim() || title.trim().length > 100) throw new Error('Título inválido')
  if (!areaId) throw new Error('Área requerida')

  // Combine date + time into scheduledAt
  const scheduledAt = new Date(`${date}T${time}:00`)

  assertDatabaseUrl()
  await db.insert(stepsActivities).values({
    userId: user.id,
    title: title.trim(),
    scheduledAt,
    scheduledDurationMinutes: duration,
    areaId,
    status: 'pending',
    planned: false,
    executorType: 'human',
  })

  revalidatePath('/calendar')
}

export async function deleteActivity(activityId: string) {
  // Validate UUID format
  const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i
  if (!UUID_REGEX.test(activityId)) throw new Error('ID inválido')

  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Unauthorized')

  assertDatabaseUrl()
  await db.delete(stepsActivities)
    .where(and(eq(stepsActivities.id, activityId), eq(stepsActivities.userId, user.id)))

  revalidatePath('/calendar')
}

export async function getAreasForUser() {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return []

  assertDatabaseUrl()
  return db.select({ id: areas.id, name: areas.name, maslowLevel: areas.maslowLevel })
    .from(areas)
    .where(eq(areas.userId, user.id))
    .orderBy(asc(areas.maslowLevel))
}
```

function validateCreateActivity(title: string, areaId: string, date: string, time: string) {
  if (!title?.trim()) return { error: 'Título requerido' }
