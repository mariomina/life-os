## Story

**As a** usuario que quiere construir rutinas sólidas en su vida,
**I want** crear hábitos con recurrencia definida (diaria, semanal, personalizada vía rrule), que el sistema genere automáticamente las occurrences pasadas/presentes como actividades en mi calendario y que detecte cuando repito espontáneamente una actividad y me sugiera convertirla en hábito,
**so that** mis rutinas estén capturadas, rastreadas y reforzadas de forma automática — sin que tenga que agendar cada repetición manualmente.

- [x] Si el área seleccionada no existe, el formulario muestra error de validación — no persiste

- [x] Un hábito editado actualiza `updated_at` y aparece con los datos nuevos

### AC5 — CRUD Hábitos: Eliminar / Desactivar

- [x] "Eliminar definitivamente" borra el registro de `habits` — las occurrences en `steps_activities` con `habit_id` set quedan con `habit_id = null` (SET NULL ON DELETE configurado en schema)
- [x] Antes de eliminar definitivamente, el sistema muestra un diálogo de confirmación indicando cuántas occurrences existentes perderán el vínculo

### AC6 — Función pura `generateHabitOccurrences`
- [x] Existe `lib/habits/occurrence-utils.ts` con función pura `generateHabitOccurrences(habit: Habit, windowStart: Date, windowEnd: Date): Date[]`
- [x] La función usa la librería `rrule` para expandir la string `habit.rrule` en el rango `[windowStart, windowEnd]`
- [x] La función retorna arreglo de fechas UTC representando cada ocurrencia en la ventana
- [x] Si la string rrule es inválida, la función lanza `Error('Invalid rrule: ...')`
- [x] La función es importable desde Server Actions y desde tests unitarios sin dependencias de DB ni browser

### AC7 — Persistencia de occurrences (pasadas/presentes)
- [x] Existe Server Action `generateAndPersistOccurrences(habitId, windowEnd)` que:
  1. Carga el hábito desde DB
  2. Calcula occurrences desde `habit.created_at` hasta `windowEnd` usando `generateHabitOccurrences`
  3. Para cada occurrence no existente en `steps_activities` (verificado por `habit_id + scheduled_at`), inserta un nuevo registro con: `habit_id`, `user_id`, `area_id`, `title = habit.title`, `description`, `executor_type = 'human'`, `planned = true`, `status = 'pending'`, `scheduled_at = occurrence`, `scheduled_duration_minutes = habit.duration_minutes`
  4. Evita duplicados: no inserta si ya existe un registro con mismo `habit_id` y `scheduled_at`
- [x] Solo se persisten occurrences `<= now()` — las futuras se calculan on-the-fly desde la rrule
- [x] La Action se llama automáticamente al crear un hábito y al editar la rrule

### AC8 — Streak calculation
- [x] Existe función pura `calculateStreak(completionDates: Date[], rrule: string): { current: number; best: number }` en `lib/habits/streak-utils.ts`
- [x] El streak actual (`streakCurrent`) se incrementa en 1 cada vez que una occurrence es marcada `completed` y la occurrence anterior (según rrule) también estaba `completed`
- [x] Si una occurrence es omitida (no completada en su fecha), el streak se resetea a 0
- [x] `streakBest` solo se actualiza si `streakCurrent > streakBest`
- [x] `lastCompletedAt` se actualiza al completar una occurrence
- [x] La lógica de streak NO está en el cliente — solo en Server Actions / pure functions

### AC9 — Detección emergente de hábito
- [x] El sistema detecta cuando un usuario registra 3 o más actividades espontáneas (`planned = false`) con el mismo título (case-insensitive, trim) en los últimos 14 días
- [x] Al detectar el patrón, muestra un banner/toast no intrusivo: "Pareces repetir '{título}' frecuentemente. ¿Quieres convertirlo en un hábito?"
- [x] El banner tiene dos acciones: "Crear hábito" (abre el formulario pre-llenado con el título) y "No, gracias" (descarta sin crear)
- [x] La detección se ejecuta en el Server Action de creación de actividad espontánea — no en el cliente
- [x] El banner solo se muestra una vez por sugerencia (no vuelve a aparecer si el usuario descartó)

### Construcción de rrule desde el formulario

```typescript
// daily a las 07:00
'FREQ=DAILY;BYHOUR=7;BYMINUTE=0;BYSECOND=0'

// weekly: lunes y miércoles a las 07:00
'FREQ=WEEKLY;BYDAY=MO,WE;BYHOUR=7;BYMINUTE=0;BYSECOND=0'
```

Para el selector de frecuencia:
- **Diaria**: `FREQ=DAILY` + hora
- **Semanal**: `FREQ=WEEKLY;BYDAY={days}` + hora
- **Personalizada**: campo de texto libre con validación via `RRule.fromString()` — si lanza, mostrar error

### Persistencia de occurrences — lógica de deduplicación

```typescript
// Verificar duplicados: buscar por habitId + scheduledAt (exacto)
const existing = await db.select({ scheduledAt: stepsActivities.scheduledAt })
  .from(stepsActivities)
  .where(and(
    eq(stepsActivities.habitId, habitId),
    gte(stepsActivities.scheduledAt, windowStart),
    lte(stepsActivities.scheduledAt, windowEnd)
  ))

const existingTimes = new Set(existing.map((r) => r.scheduledAt!.getTime()))
const toInsert = occurrences.filter((d) => !existingTimes.has(d.getTime()))
```
