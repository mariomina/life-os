## Story

**As a** usuario que ha capturado items en el Inbox,
**I want** que la IA clasifique automáticamente cada item, sugiera el área y OKR relacionados, y detecte un hueco de tiempo libre en mi calendario para ejecutarlo,
**so that** puedo convertir mis capturas en activities reales con un solo click en lugar de hacerlo manualmente.

## Acceptance Criteria

### AC1 — Migration: Campo `aiSuggestedOkrId`

**Given** el schema actual de `inbox_items` carece del campo para OKR sugerido,
**When** se aplica la migración de Story 6.2,
**Then** la tabla `inbox_items` tiene columna `ai_suggested_okr_id uuid REFERENCES okrs(id) ON DELETE SET NULL` con índice asociado, y el schema Drizzle en `lib/db/schema/inbox-items.ts` refleja este campo.

### AC2 — Acción `processInboxItem` — Happy Path IA

**Given** un item en `inbox_items` con `status = 'pending'`,
**When** se invoca la Server Action `processInboxItem(itemId)`,
**Then**:
- El item se actualiza a `status = 'processing'` antes de llamar a la IA.
- La IA recibe: `rawText`, lista de áreas activas del usuario, lista de OKRs activos (anuales + KRs del trimestre actual), y lista de huecos de tiempo libre de los próximos 7 días.
- Al completar, el item se actualiza con: `aiClassification`, `aiSuggestedAreaId`, `aiSuggestedOkrId` (nullable), `aiSuggestedSlot`, `aiSuggestedTitle`, `aiSuggestedDurationMinutes`, y `status = 'processed'`.
- Se registra `processedAt = now()`.
- Se invalida la caché de `/inbox` vía `revalidatePath`.

### AC3 — Fallback Manual (FR22)

**Given** que el proveedor de IA no está disponible (timeout, error de red, API key ausente) o lanza excepción,
**When** se invoca `processInboxItem(itemId)`,
**Then**:
- El item se actualiza a `status = 'manual'` con `aiError` conteniendo el mensaje de error legible.
- La función retorna `{ success: true, manual: true }` (no es error fatal — es fallback graceful).
- `processedAt` se registra.
- No se lanza excepción al cliente.

### AC4 — Query `getFreeSlots`

**Given** un `userId` y un rango de fechas (por defecto próximos 7 días desde hoy),
**When** se llama `getFreeSlots(userId, startDate, endDate, minDurationMinutes)`,
**Then**:
- Obtiene las activities planificadas del usuario en ese rango (via `getActivitiesForRange`).
- Calcula los huecos entre actividades, excluyendo antes de 08:00 y después de 22:00 (hora local del sistema).
- Retorna slots de duración ≥ `minDurationMinutes` (default: 30 min) como array de `{ start: Date, end: Date, durationMinutes: number }`.
- Si no hay actividades, retorna slots completos en el rango horario permitido (08:00–22:00).

### AC5 — Query `getActiveOKRsForUser`

**Given** un `userId` y año/quarter actual,
**When** se llama `getActiveOKRsForUser(userId)`,
**Then** retorna OKRs activos: los anuales del año actual + los KRs del trimestre actual, con sus campos `id`, `title`, `type`, `areaId`.

### AC6 — Extensión `ILLMProvider.classifyInboxItem`

**Given** la interfaz `ILLMProvider` en `lib/ai/types.ts`,
**When** se llama `classifyInboxItem(rawText, context)` donde `context` incluye áreas, OKRs activos y slots libres,
**Then**:
- `ClaudeProvider.classifyInboxItem` construye un prompt estructurado con el contexto y el texto.
- El modelo responde con JSON válido conteniendo: `classification`, `suggestedAreaId`, `suggestedOkrId` (nullable), `suggestedSlot` (ISO 8601), `suggestedTitle`, `estimatedDurationMinutes`.
- La implementación valida el JSON de respuesta antes de retornarlo.
- Si el JSON no es parseable, lanza error (el caller manejará el fallback).

### AC7 — Tests Unitarios

**Given** el módulo `processInboxItem` y las queries nuevas,
**When** se ejecuta `npx vitest run`,
**Then** existen tests que cubren:
- `getFreeSlots`: sin actividades retorna slots completos; con actividades rellenas retorna huecos; slots menores al mínimo son excluidos.
- `getActiveOKRsForUser`: retorna solo OKRs activos del año/quarter actual.
- Clasificación IA: mock del provider retorna clasificación válida y el item se actualiza correctamente.
- Fallback FR22: mock del provider lanza excepción y el item se marca como `manual` con `aiError`.
- Todos los tests existentes (430) siguen en verde.

```
  "suggestedOkrId": "uuid del OKR más relevante o null",
  "suggestedSlot": "ISO 8601 timestamp del primer hueco adecuado",
  "suggestedTitle": "título conciso (máx 60 chars)",
  "estimatedDurationMinutes": número entero
}
```

Usar `claude-haiku-4-5-20251001` con `max_tokens: 512` para respuesta JSON completa.

### getFreeSlots — Algoritmo

```typescript
// 1. Obtener activities en rango con getActivitiesForRange(userId, start, end)
// 2. Definir ventana horaria: 08:00–22:00 del sistema
// 3. Para cada día en el rango:
//    a. Calcular slots libres como intervalos entre actividades
//    b. Filtrar slots < minDurationMinutes
// 4. Retornar array plano de FreeSlot ordenado por start ASC
```

Se necesita una nueva variante `getActivitiesForRange` o reutilizar la lógica de `getActivitiesForWeek` con parámetros de fecha explícitos.

### processInboxItem — Flujo

```
1. Validar itemId y obtener item (debe existir + userId match)
2. UPDATE status = 'processing'
3. getFreeSlots(userId, today, today+7days)
4. getActiveAreas(userId)
5. getActiveOKRsForUser(userId)
6. try {
     result = await provider.classifyInboxItem(rawText, context)
     UPDATE item con result + status='processed' + processedAt=now()
     return { success: true, manual: false }
   } catch (err) {
     UPDATE item: status='manual', aiError=err.message, processedAt=now()
     return { success: true, manual: true }  // fallback graceful
   }
7. revalidatePath('/inbox')
```
